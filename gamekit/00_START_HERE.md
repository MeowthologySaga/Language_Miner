# Language Miner GameKit — v0.1.0-beta.1

이 폴더는 Language Miner용 `.lem` 게임을 AI와 함께 만들기 위한 제작 키트입니다.
목표는 사용자가 한 줄 아이디어만 입력해도 AI가 질문을 통해 의도를 구체화하고, 실행 가능한 PC 웹 게임 번들을 완성하도록 만드는 것입니다.

> 공개 베타 배포본에는 `13_CURRENT_RUNTIME_CONTRACT.md`가 함께 생성됩니다. 기존 예시와 현재 런타임 계약이 충돌하면 현재 런타임 계약을 우선합니다. ZIP 파일을 직접 수정하지 말고 이 원본 디렉터리를 수정한 뒤 릴리스 패키징 스크립트로 다시 생성하세요.

## 대상 사용자

- 프로그래밍을 깊게 알지 않아도 Codex, Claude, Gemini 같은 개발 AI를 사용할 수 있는 사람
- Language Miner의 게임존에 넣을 게임을 만들고 싶은 사람
- 디스코드나 링크 공유로 커뮤니티 게임을 배포하고 싶은 사람

## 기본 결과물

AI는 최종적으로 다음 파일을 만들도록 진행합니다.

```txt
my-game.lem
  manifest.json
  README.md
  security-report.md
  game/
    index.html
    main.js
    styles.css
    host-adapter.js
    mock-host.js
  assets/
    thumbnail.png
    icon.png
```

`.lem`은 zip 기반 정적 웹 게임 번들입니다. 개발 중에는 Phaser, Three.js, Vite, TypeScript, Sprite Forge를 사용할 수 있지만, 최종 제출물은 Language Miner 런처에서 바로 실행 가능한 정적 파일이어야 합니다.

## 제작 모드

AI는 구현 전에 사용자에게 제작 모드를 물어봅니다.

1. 빠른 제작
   - 질문을 적게 하고 AI가 합리적으로 가정합니다.
   - 빠르게 플레이 가능한 게임을 완성합니다.

2. 상세 제작
   - 질문을 많이 해서 게임성, 재화 흐름, 다이아 사용처, 에셋 방향을 충분히 고정합니다.
   - 어비스 서머너급 완성도를 목표로 합니다.

### 제작 모드 질문 게이트

AI는 사용자의 첫 요청이 아무리 짧거나 명령형이어도, 구현 전에 반드시 제작 모드를 질문하고 답변을 기다립니다.

다음 표현은 "질문 생략 허가"가 아니라 아이디어 입력으로만 해석합니다.

- 뱀서류 만들자
- 방치형 게임 만들어줘
- 카드 게임 하나 만들어
- 알아서 만들어
- 빠르게 해줘
- 바로 만들어줘

AI가 제작 모드 질문을 생략하고 빠른 제작 모드로 진행할 수 있는 경우는 사용자가 명시적으로 다음 중 하나를 말한 경우뿐입니다.

- "질문 없이 바로 만들어"
- "빠른 제작 모드로 진행해"
- "모든 선택은 AI가 알아서 하고 구현해"
- "제작 모드 질문 생략"

그 외에는 반드시 아래 질문을 먼저 합니다.

```md
제작 모드를 선택해주세요.

1. 빠른 제작
   - 핵심 질문만 하고 빠르게 플레이 가능한 게임을 만듭니다.
   - 세부 기획은 AI가 합리적으로 가정합니다.

2. 상세 제작
   - 게임성, 다이아 소비처, 에셋 방향, 장기 성장 구조를 먼저 정하고 만듭니다.
   - 더 완성도 높은 장기 플레이 게임을 목표로 합니다.

직접 원하는 방식이 있으면 자유롭게 적어도 됩니다.
```

## 핵심 원칙

- 게임은 PC 16:9 실행을 기본으로 합니다.
- 모바일 대응은 필수가 아닙니다.
- 게임은 별도 창에서 실행되는 런처 게임으로 가정합니다.
- 외부 네트워크는 기본 차단입니다.
- 게임은 다이아를 지급할 수 없습니다.
- 게임은 다이아 소비 요청만 할 수 있습니다.
- 모든 다이아 소비는 Language Miner 런처 확인 모달을 거쳐야 합니다.
- 게임 저장은 `window.LEM_GAME_HOST_API.save.load/write/clear`를 사용합니다.
- `localStorage`는 테스트 환경 fallback으로만 사용합니다.
- 사용자 PC 파일 시스템에 직접 접근하는 코드는 작성하지 않습니다.

## AI 작업 순서

AI는 다음 순서를 지킵니다.

```txt
1. 사용자 한 줄 아이디어 분석
2. 빠른 제작 / 상세 제작 모드 선택 질문
3. 사용자 답변 대기
4. 제작 모드에 맞는 추가 질문
5. 장르 레시피 선택
6. Game Design Contract 작성
7. Diamond Economy Contract 작성
8. LEM manifest 초안 작성
9. 에셋 제작 계획 작성
10. 비주얼 검토 보드 생성
11. 사용자 비주얼 승인 대기
12. 구현
13. 자동 테스트와 플레이 검증
14. README, 썸네일, 설명 작성
15. .lem 패키징
16. 최종 리포트 작성
```

## 비주얼 승인 게이트

AI는 구현 전에 인게임 느낌과 주요 화면 흐름을 이미지로 보여주고 사용자의 확인을 받아야 합니다.

필수 보드:

- 인게임 느낌 보드: 실제 플레이 화면의 분위기, HUD, 핵심 액션, 보상 감각을 보여줍니다.
- 씬 플로우 보드: 메인 메뉴, 선택 화면, 준비 화면, 인게임, 성장 선택, 보스/목표, 결과, 장기 성장 사이의 흐름을 보여줍니다.

게임이 매우 단순한 경우에도 최소 1장의 비주얼 보드는 생성합니다. 여러 씬이 있는 게임은 씬 플로우 보드를 반드시 생성합니다.

AI는 보드를 보여준 뒤 아래처럼 확인합니다.

```md
이 이미지 방향이 정말 원하는 게임이 맞나요?
수정할 분위기, 화면 흐름, UI, 보상 구조가 있으면 말해주세요.
확인되면 이 방향으로 구현을 시작하겠습니다.
```

사용자가 확인하기 전에는 파일 생성, 코드 작성, 패키징을 시작하지 않습니다.

## 직접 복제 금지

어비스 서머너는 컨셉 참고 자료가 아닙니다. 다음만 참고합니다.

- 게임 루프 밀도
- 보상과 성장 연결 방식
- 다이아를 쓰고 싶게 만드는 선택 구조
- 저장, 설정, 상점, 수집, 성장 같은 완성도 기준
- README와 런처용 설명 구성

다음은 복제하지 않습니다.

- 세계관
- 캐릭터
- 몬스터
- 아이템명
- UI 문구
- 어두운 고딕 판타지 분위기
- 동일한 밸런스 수치
