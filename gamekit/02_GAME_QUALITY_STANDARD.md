# Game Quality Standard

Language Miner 게임은 단순한 클릭 데모가 아니라 사람들이 실제로 플레이하고 싶어지는 게임이어야 합니다.

## 기본 완성 기준

최종 게임은 최소한 다음 조건을 만족해야 합니다.

- PC 16:9 화면에서 바로 플레이 가능
- 첫 10초 안에 무엇을 하는 게임인지 이해 가능
- 1분 안에 첫 보상 또는 성장 선택 발생
- 5분 안에 새 시스템, 새 목표, 새 보상 중 하나 등장
- 저장 기능 존재
- 설정 또는 초기화 기능 존재
- 튜토리얼, 도움말, 안내 문구 중 하나 존재
- 반복 플레이 이유 존재
- 다이아 소비처 3종 이상 존재
- Sprite Forge 또는 생성 에셋 기반 주요 시각 자산 3종 이상 존재
- 런처용 썸네일, 짧은 설명, 장르 태그 제공
- README 제공
- 크기가 달라지는 주요 버튼/패널/칩 UI가 9-slice 또는 그에 준하는 확장 가능한 프레임으로 구현되어 깨지지 않음

## 게임다운 느낌 체크리스트

AI는 구현 후 다음 질문에 답해야 합니다.

```md
1. 플레이어가 지금 무엇을 해야 하는지 바로 보이는가?
2. 보상이 너무 느리거나 너무 의미 없지는 않은가?
3. 성장 선택이 1개뿐이라 단조롭지는 않은가?
4. 다이아를 쓸 이유가 게임 루프 안에 자연스럽게 있는가?
5. 다이아가 없어도 기본 플레이는 재미있는가?
6. 저장 후 다시 켜도 진행이 유지되는가?
7. 런처에서 봤을 때 어떤 게임인지 썸네일과 설명으로 알 수 있는가?
8. 에셋이 게임 분위기와 맞는가?
9. 플레이가 막히거나 아무 일도 안 일어나는 시간이 긴가?
10. README만 보고 실행 방법을 알 수 있는가?
```

## 장기 플레이 기준

오래 할 수 있는 게임은 다음 중 최소 2개 이상을 가져야 합니다.

- 캐릭터/유닛 수집
- 스킨/의상 수집
- 장비/무기 성장
- 스테이지/층/지역 진행
- 도감/업적
- 일일 도전 또는 반복 도전
- 리셋/환생/전승형 장기 성장
- 빌드 다양성
- 랭크/점수 갱신

## 짧은 세션 게임 기준

짧은 게임은 로그라이크식 반복 구조를 권장합니다.

- 한 판은 5~20분
- 매 판 보상 또는 해금이 남음
- 캐릭터, 무기, 유물, 스킨 같은 장기 수집 요소 존재
- 다이아는 입장권, 부활, 캐릭터, 스킨, 보너스 선택지 등에 사용 가능
