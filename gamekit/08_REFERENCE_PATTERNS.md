# Reference Patterns

이 문서는 특정 게임의 설정을 복제하지 않고, 완성도 높은 게임 구조만 추출하기 위한 참고 자료입니다.

## 참조 허용

- 보상과 성장의 연결 방식
- 짧은 시간 안에 재미가 보이는 구조
- 다이아를 쓰고 싶어지는 순간 설계
- 장기 성장과 수집 구조
- 저장, 설정, 상점, 도감, 튜토리얼 같은 완성도 요소
- README와 런처 설명 구성

## 참조 금지

- 기존 게임의 세계관
- 캐릭터명
- 적 이름
- 아이템명
- 스킬명
- UI 문구
- 시각 분위기
- 동일한 밸런스 수치

## 성장형 게임 공통 루프

```txt
핵심 행동
→ 보상 획득
→ 성장 선택
→ 더 어려운 목표 도전
→ 새 수집 요소 획득
→ 누적 보너스 증가
→ 정체 시 장기 성장 또는 새 전략 사용
```

## 다이아 유도 루프

```txt
무료 플레이로 재미 체감
→ 조금 더 빨리/넓게/예쁘게 하고 싶은 욕구 발생
→ 다이아 소비처 노출
→ 공부해서 다이아 획득
→ 게임에서 의미 있는 선택으로 소비
→ 성장 체감
```

## 좋은 소비처 패턴

- 시간을 줄인다.
- 선택지를 늘린다.
- 수집 욕구를 자극한다.
- 실패를 한 번 완화한다.
- 외형 만족감을 준다.
- 진행을 편하게 한다.

## 피해야 할 소비처 패턴

- 기본 재미를 잠근다.
- 튜토리얼을 막는다.
- 너무 자주 결제를 요구한다.
- 무료 플레이어가 아무것도 못 하게 만든다.
- 사용자가 무슨 효과인지 모르게 만든다.
