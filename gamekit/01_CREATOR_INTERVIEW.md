# Creator Interview

AI는 구현 전에 사용자의 의도를 충분히 묻습니다. 한 줄 아이디어만 보고 바로 만들면 게임성이 약해지기 쉽기 때문입니다.

## 첫 질문

이 질문은 사용자가 제작 모드 질문 생략을 명시하지 않은 모든 경우에 반드시 먼저 합니다.

```md
제작 모드를 골라주세요.

1. 빠른 제작
   - 핵심 질문만 하고 바로 만듭니다.
   - 세부 기획은 AI가 많이 가정합니다.

2. 상세 제작
   - 게임성, 다이아 구조, 성장 루프, 에셋 방향까지 자세히 물어봅니다.
   - 완성도 높은 장기 플레이 게임을 목표로 합니다.

직접 원하는 방식이 있으면 자유롭게 적어도 됩니다.
```

## 빠른 제작 질문

빠른 제작은 최대 5개만 묻습니다.

```md
1. 어떤 게임을 만들고 싶나요? 한 문장으로 적어주세요.
2. 플레이어가 가장 많이 반복할 행동은 무엇인가요?
3. 분위기는 어떤 쪽인가요? 예: 귀여움, 진지함, 액션, 힐링, 어두움
4. 다이아를 쓰고 싶어지는 순간은 무엇이면 좋나요?
5. 피하고 싶은 소재나 게임 방식이 있나요?
```

사용자가 답하지 않으면 AI가 합리적으로 가정하고 진행합니다.

## 상세 제작 질문

상세 제작은 필요한 만큼 질문하되, 한 번에 너무 많이 묻지 않습니다. 우선 8~12개를 묻고, 부족하면 추가 질문합니다.

```md
1. 한 줄 아이디어를 조금 더 설명해주세요.
2. 플레이어가 조작하는 대상은 무엇인가요?
3. 핵심 반복 행동은 무엇인가요?
4. 1분 안에 어떤 보상을 받아야 하나요?
5. 5분 안에 어떤 새 시스템이 열리면 좋나요?
6. 장기 목표는 무엇인가요?
7. 실패/정체 상황은 어떻게 생기나요?
8. 다이아 소비처는 어떤 쪽이 어울리나요?
   - 캐릭터 뽑기
   - 의상/스킨 뽑기
   - 무기/장비 상자
   - 부활
   - 입장권
   - 성장 가속
   - 편의 기능
9. 플레이어가 공부해서 다이아를 벌고 싶어지는 이유는 무엇이어야 하나요?
10. 원하는 그래픽 톤은 무엇인가요?
11. Sprite Forge로 꼭 만들고 싶은 에셋이 있나요?
12. 절대 넣고 싶지 않은 소재, 분위기, BM이 있나요?
```

## 의도 고정 질문

AI는 설계 전에 반드시 다음을 사용자에게 확인합니다.

```md
제가 이해한 게임은 다음과 같습니다.

- 장르:
- 핵심 재미:
- 주요 반복 루프:
- 장기 성장:
- 다이아 사용처:
- 에셋 방향:
- 런처 표시 문구:

이 방향으로 고정하고 구현해도 될까요?
```

이 단계에서 사용자가 "알아서 해"라고 하면 AI는 현재 이해를 기준으로 진행합니다.
단, 첫 입력의 "알아서 만들어"는 제작 모드 질문 생략 허가로 보지 않습니다.

## 비주얼 확인 질문

AI는 Game Design Contract와 Diamond Economy Contract를 고정한 뒤, 구현 전에 비주얼 검토 보드를 생성하고 사용자에게 다시 확인합니다.

필수 확인 문장:

```md
이 이미지 방향이 정말 원하는 게임이 맞나요?
수정할 분위기, 화면 흐름, UI, 보상 구조가 있으면 말해주세요.
확인되면 이 방향으로 구현을 시작하겠습니다.
```

사용자가 "맞아", "이대로", "좋아", "진행해"처럼 승인한 뒤에만 구현을 시작합니다.
