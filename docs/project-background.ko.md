# Language Miner 프로젝트 배경

[English](project-background.en.md) · [메인 소개](../README.md)

Language Miner는 처음부터 공개 서비스를 목표로 만든 프로젝트가 아니었습니다. 영어를 읽고 들으며 발견한 표현을 카드로 만들고, 복습한 뒤 실제 문장으로 다시 써 보고 싶어서 시작한 개인용 도구였습니다.

## 개인 도구에서 오픈소스로

개인에게 맞춘 기능이 늘면서 다른 학습자도 비슷한 문제를 겪는다는 생각이 들었습니다. 오픈소스 기여자를 지원한다는 OpenAI 관련 소식을 접한 일도 소스를 공개하는 방향을 진지하게 검토한 계기 중 하나였습니다. 특정 혜택을 받았다는 의미는 아니며, 프로젝트의 사용 조건이나 방향을 결정하는 후원 관계도 아닙니다.

공개하기로 마음먹은 뒤에는 “내 컴퓨터에서만 돌아가면 된다”는 기준을 버려야 했습니다. 개인 설정, API 키, 로컬 경로, 실험 로그와 라이선스가 불명확한 자산을 정리하고, 처음 설치한 사람이 배경지식 없이도 안전하게 사용할 수 있는 흐름이 필요해졌습니다.

## 왜 문장 카드인가

단어장을 오래 봐도 실제 대화에서 문장이 바로 나오지 않는 경험이 출발점이었습니다. 단어의 뜻을 알아보는 기억과, 특정 상황에서 자연스러운 문장을 꺼내는 기억은 같지 않습니다.

그래서 Language Miner의 중심은 다음 연결입니다.

```text
표현 발견 → 문장 카드 → 간격 복습 → 영작·대화에서 사용
```

읽기와 듣기만으로 끝내지 않고, 말하기와 쓰기로 같은 표현을 다시 꺼내게 하는 것이 핵심입니다.

## UGC로 넓어진 이유

캐릭터챗은 배운 문장을 상황 속에서 써 보는 방법으로 시작했습니다. PlayZone은 학습에서 얻은 로컬 보상과 표현을 작은 게임 안에서 활용해 학습 루프에 재미와 제작자 참여를 더하려는 시도입니다.

UGC가 늘려면 하나의 웹 서비스가 모든 파일을 소유할 필요는 없습니다. 제작자는 자신의 GitHub Release나 외부 저장소에 콘텐츠를 두고, Discord는 발견과 커뮤니티를 담당할 수 있습니다. 앱은 가져온 팩에 같은 기술 검사를 적용합니다.

이 구조를 선택한 이유는 운영자가 사용자 콘텐츠와 공용 API 키를 보관하는 중앙 서버를 만들지 않고도 커뮤니티가 자랄 수 있게 하기 위해서입니다. 동시에 링크 하나가 안전이나 저작권을 보증하지 않는다는 한계도 분명히 안내합니다.

## 로컬 우선 원칙

- 새 설치에서는 AI를 연결하지 않습니다.
- 가능한 작업은 로컬 데이터와 로컬 모델부터 사용합니다.
- 클라우드 AI는 사용자의 키와 명시적인 전송 동의로만 켭니다.
- 개발자 소유 광고, 분석, 텔레메트리 서버를 두지 않습니다.
- UGC는 실행 전에 파일·권한·출처 계약을 확인합니다.

로컬 우선은 “아무 데이터도 위험하지 않다”는 뜻이 아닙니다. 로컬 평문 데이터는 Windows 계정에 접근하는 사람과 악성 프로그램에 노출될 수 있고, 원격 Ollama 주소도 외부 전송입니다. 사용자가 실제 경계를 이해할 수 있도록 설명하는 것까지 제품의 일부로 봅니다.

## 첫 공개 베타의 범위

첫 목표는 기능 수를 늘리는 것이 아니라 현재 보이는 학습 루프를 안전하고 이해하기 쉽게 공개하는 것입니다.

- Windows 10/11 x64 설치판과 포터블판;
- 한국어·영어 UI;
- 로컬 우선 카드·복습·입출력 학습;
- 선택형 AI와 비용 안내;
- 전체 백업·복원;
- 검증된 캐릭터·게임 UGC;
- 정적 GitHub Pages 문서와 공개 개발 과정.

Anki/CSV 내보내기, UGC 카탈로그, Chrome Web Store, 자동 업데이트, macOS, 일본어·중국어 UI와 전체 웹앱은 첫 베타 이후에 실제 사용을 보고 다시 판단합니다.

## 프로젝트가 지향하는 것

Language Miner가 모든 사람에게 같은 공부법을 강요하는 도구가 되기를 바라지 않습니다. 학습자가 자기 문장을 모으고, 자기 속도로 복습하고, 안전 경계를 이해한 채 대화와 놀이로 다시 사용할 수 있는 기반을 지향합니다. 제작자에게는 중앙 서비스의 허락 없이도 캐릭터와 게임을 만들고 버전과 라이선스를 직접 관리할 수 있는 열린 형식을 제공하려 합니다.
