# Language Miner 완전 사용 설명서

[English](complete-user-manual.en.md) · [간단 사용 가이드](user-guide.ko.md) · [Windows 설치](install-windows.ko.md) · [개인정보 안내](privacy.ko.md) · [UGC 정책](ugc-policy.ko.md)

이 설명서는 영어를 공부하지만 웹·API·로컬 AI에는 익숙하지 않은 사용자를 위한 튜토리얼형 안내서입니다. 메뉴를 나열하는 데서 끝내지 않고, **어디를 누르는지 → 왜 쓰는지 → 무엇이 보이면 성공인지 → 안 되면 무엇을 확인할지** 순서로 설명합니다.

이 설명서는 공개 베타 [v0.1.0-beta.1](https://github.com/MeowthologySaga/Language_Miner/releases/tag/v0.1.0-beta.1)을 기준으로 합니다. 파일 목록, 알려진 문제와 변경사항은 해당 Release 안내가 이 문서보다 우선합니다.

## 1. 먼저 알아둘 말

- **UGC(User-Generated Content)**: 사용자가 만들어 공유하는 캐릭터팩이나 게임팩입니다.
- **BYOK(Bring Your Own Key)**: 앱 개발자의 공용 키가 아니라, 사용자가 공급자에게 직접 발급받은 API 키를 넣어 쓰는 방식입니다. 과금 관계도 사용자와 공급자 사이에 생깁니다.
- **loopback(루프백)**: `localhost`, `127.0.0.1`, `::1`처럼 요청이 같은 PC로 되돌아오는 주소입니다. Ollama는 이 주소일 때만 앱에서 “로컬”로 취급합니다.
- **NSIS**: 일반적인 Windows 설치 프로그램 형식입니다. Language Miner의 설치판이 이 형식을 사용합니다.
- **SBOM**: 배포물에 어떤 소프트웨어 부품이 들어 있는지 적은 기계 판독용 목록입니다.
- **CSP(Content Security Policy)**: 게임 페이지가 불러오거나 실행할 수 있는 대상을 제한하는 브라우저 보안 규칙입니다.
- **quota(쿼터)**: 공급자가 정한 요청·토큰 상한입니다. 앱의 비용 추정치와 달리 공급자 쪽에서 실제 요청을 거절할 수 있습니다.

## 2. 왜 단어장이 아니라 문장 루프인가

`late = 늦은`만 외우면 글에서 알아볼 수는 있어도, 약속에 늦는 순간 바로 문장이 나오지 않을 수 있습니다. 다음처럼 실제 문장으로 기억하면 단어, 문법, 자연스러운 결합, 상황을 함께 꺼낼 수 있습니다.

> I’m running a little late.

Language Miner의 기본 루프는 다음과 같습니다.

1. 읽거나 듣다가 실제 표현을 발견합니다.
2. 표현이 들어 있는 문장 전체를 읽기·듣기·말하기 카드 중 알맞은 형태로 저장합니다.
3. 간격 반복으로 잊기 직전에 다시 떠올립니다.
4. 영작이나 캐릭터 대화에서 같은 표현을 직접 꺼내 씁니다.

읽기·듣기 같은 **인풋**은 영어를 알아보는 힘을 만들고, 말하기·영작 같은 **아웃풋**은 필요한 순간에 영어를 꺼내는 힘을 만듭니다. 어느 한쪽만으로는 부족하기 때문에 앱도 두 방향을 한 루프로 연결합니다.

[![표현 발견, 카드, 복습, 영작과 대화로 이어지는 학습 루프](site/assets/app-images…17910 tokens truncated…tension options showing the offline pending-item count and the Delete all pending items button](site/assets/app-images/en/29-extension-queue-clear.webp)

*그림 1. 같은 문장을 발견하고, 저장하고, 복습하고, 실제 상황에서 다시 쓰는 흐름입니다. GIF가 무거우면 클릭해 원본을 따로 열 수 있습니다.*

## 3. 설치와 첫 실행

### 3.1 공식 Release에서 설치 파일 고르기

**경로:** [v0.1.0-beta.1 Release](https://github.com/MeowthologySaga/Language_Miner/releases/tag/v0.1.0-beta.1) → Assets

**쓰는 이유:** 출처와 파일이 맞는지 확인한 뒤 설치하기 위해서입니다.

**따라 하기**

1. 같은 Release에서 `Language-Miner-Setup-0.1.0-beta.1-x64.exe` 또는 `Language-Miner-Portable-0.1.0-beta.1-x64.exe`, `SHA256SUMS.txt`, SBOM을 받습니다.
2. 평소 Windows 앱처럼 쓰려면 설치판, 설치 과정 없이 실행 파일을 직접 관리하려면 포터블판을 고릅니다.
3. [Windows 설치 안내](install-windows.ko.md)에 따라 SHA-256을 비교합니다.
4. 첫 베타는 서명되지 않을 수 있으므로 SmartScreen에 표시된 게시자와 파일 출처를 다시 확인합니다.
5. 확신할 수 없으면 실행하지 않습니다.

**완료 상태:** 앱 창이 열리고 첫 실행 언어 화면이 보입니다.

**막히면:** 파일명이 비슷하더라도 다른 사이트에서 다시 받지 말고 공식 Release로 돌아갑니다. 설치판을 다시 실행하면 같은 사용자·같은 경로에서는 보통 업그레이드 또는 복구 설치가 되지만, 버전의 Release 안내를 먼저 확인하세요. 포터블은 “설치 과정이 없다”는 뜻이며 학습 데이터까지 실행 파일 옆에만 남는다는 뜻은 아닙니다.

**개인정보·비용:** 설치 자체에는 AI API 비용이 없습니다. Windows 제거 프로그램은 사용자 학습 데이터를 남길 수 있으므로 PC를 넘기기 전에는 앱 안의 전체 삭제를 먼저 사용합니다.

### 3.2 앱 언어, 모국어, 학습 언어 고르기

**경로:** 첫 실행 팝업 → 언어 설정

**쓰는 이유:** 메뉴 언어와 학습 기준 언어를 분리하기 위해서입니다. 앱 언어를 영어로 바꿔도 카드의 모국어·학습 언어가 자동으로 바뀌지 않습니다.

**따라 하기**

1. `앱 언어`에서 메뉴와 안내 문구의 언어를 고릅니다.
2. `모국어`에서 뜻과 설명의 기준 언어를 고릅니다.
3. `학습 언어`에서 카드와 연습에 쓸 언어를 고릅니다.
4. 한국어 화자가 영어를 공부한다면 `한국어 / 한국어 / English`를 선택합니다.
5. 마지막 AI 안내에서는 키를 넣지 않고 `앱 둘러보기`로 진행해도 됩니다.

**완료 상태:** 오늘 화면이 열리고 같은 팝업이 다음 실행부터 반복해서 나타나지 않습니다.

**막히면:** 팝업이 매번 다시 뜬다면 앱을 완전히 종료한 뒤 다시 실행하고, 설치판과 포터블판을 번갈아 실행하고 있지 않은지 확인합니다. 계속되면 일반 버그로 보고하세요.

**개인정보·비용:** 언어 선택은 로컬 설정입니다. 이 단계에서 외부 AI 요청은 없어야 합니다.

[![첫 실행에서 앱 언어, 모국어, 학습 언어를 고르는 화면](site/assets/app-images/ko/01-onboarding-language.webp)](site/assets/app-images/ko/01-onboarding-language.webp)

## 4. 10분 시작 순서

이 절은 “10분 안에 앱을 다 익힌다”는 약속이 아니라, 설정에 갇히지 않고 첫 학습 루프를 시작하는 순서입니다. 처음에는 AI를 연결하지 않아도 됩니다.

### 4.1 튜토리얼과 실제 화면을 구분하기

**경로:** 관리 → 튜토리얼

**쓰는 이유:** 실수 걱정 없이 선택, 카드 생성, 복습 조작을 연습하기 위해서입니다.

튜토리얼은 실제 화면을 닮은 **연습용 복제 공간**입니다. 여기서 누른 동작은 튜토리얼 진행만 기록하며 실제 카드 목록을 바꾸지 않습니다. 실제 카드를 만들려면 튜토리얼을 나와야 합니다.

**완료 상태:** 화면에 “이곳은 튜토리얼입니다”라는 안내가 보입니다.

**막히면:** 실제 카드 목록에서 새 카드가 안 보인다면 튜토리얼 안에서 만든 것이 아닌지 먼저 확인합니다.

**개인정보·비용:** 튜토리얼 샌드박스는 외부 AI를 호출하지 않는 연습 흐름입니다.

[![실제 카드와 분리된 튜토리얼 샌드박스](site/assets/app-images/ko/manual-01-tutorial-sandbox.webp)](site/assets/app-images/ko/manual-01-tutorial-sandbox.webp)

*튜토리얼 안내와 연습용 네비게이션이 함께 보입니다. 이 화면에서의 조작은 실제 카드 목록을 바꾸지 않습니다.*

### 4.2 실제 내장 글에서 문장 고르기

**경로:** 읽고 듣기 · Input → 읽기 → 웹 리더 → `Language Miner 연습 글`

연습 글의 내부 주소는 `lem-practice://reader/getting-started`입니다. 앱에 들어 있는 글이므로 외부 사이트에 접속하지 않습니다.

1. `running a little late`를 드래그합니다.
2. 뜨는 도구에서 `문장카드`를 누릅니다.
3. 전체 문장과 가까운 문맥이 후보에 들어갔는지 확인합니다.

**완료 상태:** `I’m running a little late.`의 읽기 카드 미리보기가 열립니다.

**막히면:** 드래그 직후가 아니라 `문장카드` 동작을 눌렀는지 확인합니다. 선택이 어긋났다면 `다시 선택`을 누릅니다.

**개인정보·비용:** 내장 연습 글 자체는 외부 전송이 없습니다. Gemini나 원격 Ollama를 카드 엔진으로 골랐다면 실제 생성 직전 별도 전송 확인이 나옵니다.

[![웹 리더 내장 연습 글에서 running a little late를 선택한 화면](site/assets/app-images/ko/06-web-reader-select.webp)](site/assets/app-images/ko/06-web-reader-select.webp)

*그림 3. 실제 웹 리더에서 표현을 선택해 읽기 카드 후보를 만듭니다.*

### 4.3 카드 본문을 고친 뒤 저장하기

**경로:** 읽기 카드 미리보기 → 편집 → 카드 저장

1. 영어 문장, 직역, 자연스러운 뜻, 학습 메모를 읽습니다.
2. AI가 만든 설명이 내 의도와 다르면 미리보기의 `편집` 또는 `본문 편집`에서 고칩니다.
3. `카드 저장` 또는 화면에 표시된 저장 동작을 누릅니다.

**완료 상태:** 저장 완료 안내가 나오고 카드 목록 또는 복습에서 카드가 보입니다.

**막히면:** 생성 결과를 그대로 믿지 말고 문장 언어, 선택 범위, 번역을 먼저 고칩니다. 저장된 카드 화면은 현재 태그 편집과 삭제 중심이므로, 본문은 **저장 전 미리보기에서** 확정하는 것이 안전합니다.

**개인정보·비용:** 카드 내용은 로컬 SQLite 등에 평문으로 저장됩니다. 공용 PC에서는 개인 대화나 회사 문장을 카드로 만들지 않는 편이 안전합니다.

[![I’m running a little late 읽기 카드의 본문을 확인하고 저장하는 미리보기](site/assets/app-images/ko/07-reading-card-preview.webp)](site/assets/app-images/ko/07-reading-card-preview.webp)

*그림 4. 생성된 뜻과 메모를 직접 확인하고 고친 뒤 저장합니다.*

### 4.4 첫 복습과 첫 영작

**경로:** 복습 → 읽기 카드 → 복습하기

1. 답을 보기 전에 문장의 뜻과 상황을 떠올립니다.
2. `답 보기`를 누릅니다.
3. 어렵게 떠올렸으면 `다시` 또는 `어려움`, 바로 떠올렸으면 `좋음` 또는 `쉬움`을 고릅니다.
4. 말하고 쓰기 · Output → 영작 훈련에서 “친구에게 조금 늦는다고 알리기”와 같은 한국어 상황을 보고 문장을 직접 씁니다.

**완료 상태:** 복습을 평가하면 다음 복습 시점이 조정되고, 영작 결과에서 내 답안·표현 체크·추천 답안을 비교할 수 있습니다.

**막히면:** 카드가 아직 복습 예정이 아니면 복습 화면의 덱별 대기 수를 확인합니다. 영작할 카드가 없다면 먼저 읽기 또는 말하기 카드를 저장합니다.

**개인정보·비용:** 복습 일정과 기본 영작 비교는 로컬 데이터입니다. 영작 결과는 다음 연습에서 무엇을 더 집중할지 판단하는 자료이며, SRS 카드 힌트를 몰래 자동 변경하지 않습니다.

## 5. 오늘 화면, 루틴, 미션, 다이아

### 5.1 오늘 작업대 읽기

**경로:** 오늘

**쓰는 이유:** 복습 예정량, Life Mining 후보, 듣기 문장을 한 화면에서 보고 다음 행동을 하나만 고르기 위해서입니다.

상단 요약 카드는 `오늘 복습`, `새 카드 후보`, `듣기 루프`로 나뉩니다. 아래에는 최근 공부 잔디와 오늘 루틴이 있습니다.

**예시:** 복습이 8장이고 Life Mining 후보가 2개라면, 먼저 `복습 시작`으로 예정 카드를 처리한 뒤 후보를 확인합니다.

**완료 상태:** 각 카드의 수와 진행 상태가 실제 데이터에 맞게 바뀝니다.

**막히면:** 숫자가 갱신되지 않으면 해당 화면을 다시 열거나 오늘 화면의 동작을 다시 눌러 봅니다. 앱 전체를 새로고침해야 하는 것이 정상 동작은 아닙니다.

**개인정보·비용:** 요약과 잔디는 로컬 활동 기록입니다. 개발자 소유 분석 서버로 보내지 않습니다.

[![오늘 학습 허브의 요약과 활동 기록](site/assets/app-images/ko/manual-02-today-hub.webp)](site/assets/app-images/ko/manual-02-today-hub.webp)

*복습·후보·듣기 요약과 활동 기록을 한 화면에서 확인합니다.*

### 5.2 오늘 루틴 이어가기

**경로:** 오늘 → 오늘 루틴 시작

루틴 순서는 `복습 → 듣기 루프 → 영작 훈련 → 보상 정리`입니다. 다른 메뉴로 나갔다 돌아와도 `오늘 루틴 이어하기`로 다음 단계에 복귀할 수 있습니다. 필요하면 건너뛴 단계를 다시 열 수 있습니다.

**완료 상태:** 마지막 보상 정리까지 진행하면 오늘 루틴 완료로 표시됩니다.

**막히면:** `현재 단계 열기`가 반응하지 않으면 해당 기능을 왼쪽 메뉴에서 직접 열고 완료한 뒤 오늘 화면으로 돌아옵니다.

**개인정보·비용:** 루틴 자체에는 비용이 없습니다. 루틴 안에서 클라우드 카드 생성이나 번역을 시작할 때만 별도 사전 확인이 필요합니다.

[![진행 중인 오늘 루틴과 현재 단계](site/assets/app-images/ko/manual-03-today-routine.webp)](site/assets/app-images/ko/manual-03-today-routine.webp)

*현재 단계와 다음 동작을 확인한 뒤 중단한 위치에서 이어갑니다.*

### 5.3 미션을 끝내고 다이아 받기

다이아는 현금이나 암호화폐가 아니라 **이 기기의 로컬 학습 보상**입니다. 거래하거나 환전할 수 없습니다. 진행만 채워서는 잔액이 늘지 않을 수 있으므로 완료한 미션의 `받기`를 눌러야 합니다.

| 오늘 미션 | 목표 | 보상 |
|---|---:|---:|
| 읽기 카드 만들기 | 5장 | 15 다이아 |
| 듣기 루프 문장 완료 | 30문장 | 20 다이아 |
| 영작 확인 완료 | 3회 | 15 다이아 |
| Life Mining 카드 만들기 | 5장 | 25 다이아 |
| 읽기 카드 덱 복습 완료 | 1회 | 15 다이아 |
| 듣기 카드 덱 복습 완료 | 1회 | 15 다이아 |
| 말하기 카드 덱 복습 완료 | 1회 | 15 다이아 |
| 위 기본 미션 보상 전부 수령 | 오늘 보너스 | 30 다이아 |

**완료 상태:** 미션이 `수령 완료`로 바뀌고 상단 다이아 잔액과 오늘 획득량이 증가합니다.

**막히면:** 목표를 달성했는데 받을 수 없다면 오늘 날짜와 프로필을 확인한 뒤 해당 활동을 한 번 더 완료합니다. 중복 클릭으로 같은 보상이 두 번 들어가는 것이 정상은 아닙니다.

**개인정보·비용:** 다이아 사용에는 실제 결제가 연결되지 않습니다. PlayZone에서 다이아를 쓸 때는 게임이 선언한 동작·금액·사유가 확인창과 일치해야 합니다.

[![일일 미션 진행률과 다이아 보상](site/assets/app-images/ko/manual-04-daily-missions.webp)](site/assets/app-images/ko/manual-04-daily-missions.webp)

*완료한 미션의 `받기`를 눌러야 로컬 다이아 잔액에 반영됩니다.*

## 6. 프로필과 앱 언어

### 6.1 프로필 만들기·전환·복제

**경로:** 설정 → 일반 → 현재 학습 프로필 → 프로필 관리

**쓰는 이유:** 영어 학습과 다른 언어 학습, 또는 서로 다른 목표를 카드·복습 기록과 함께 분리하기 위해서입니다.

1. `새 프로필`을 누릅니다. 현재 언어 설정을 복사해 시작합니다.
2. 이름, 배우려는 언어, 모국어를 확인합니다.
3. 비슷한 구성이 필요하면 `복제`를 사용합니다.
4. 상단 프로필 버튼이나 설정의 `프로필 전환`에서 바꿉니다.

**완료 상태:** 상단에 새 프로필 이름과 `학습 언어 · 모국어 기준` 요약이 보이고 카드·복습 수가 해당 프로필 기준으로 바뀝니다.

**막히면:** 기본 프로필은 삭제할 수 없습니다. 다른 프로필로 전환하거나 기본 프로필 안의 데이터를 개별 삭제하세요.

**개인정보·비용:** 카드, 복습, 듣기 진행, 문서·내보내기, 번역 캐시는 프로필별입니다. Life Log 원문은 장치 공용이며 프로필별 처리 상태만 나뉩니다. 프로필 삭제는 되돌릴 수 없습니다.

[![학습 언어별 프로필을 관리하는 화면](site/assets/app-images/ko/manual-05-profile-manager.webp)](site/assets/app-images/ko/manual-05-profile-manager.webp)

*현재 프로필을 확인하고 새 프로필 생성·복제·언어 설정을 관리합니다.*

### 6.2 앱 표시 언어 바꾸기

**경로:** 설정 → 일반 → 앱 표시 언어

`한국어`와 `English`를 즉시 바꿀 수 있습니다. 이는 메뉴와 안내만 바꾸며 학습 프로필의 언어에는 영향을 주지 않습니다.

**완료 상태:** 현재 화면의 메뉴와 라벨이 선택한 언어로 바뀌고 설정이 자동 저장됩니다.

**막히면:** 일부 문구만 예전 언어로 남으면 화면을 다시 열고, 계속 남는다면 번역 누락 버그로 신고합니다.

**개인정보·비용:** 로컬 표시 설정이며 외부 요청이 없습니다.

## 7. 세 가지 카드와 카드 관리

### 7.1 카드 종류 고르기

| 카드 | 앞에서 먼저 접하는 것 | 훈련 목표 | 주로 만드는 곳 |
|---|---|---|---|
| 읽기 카드 | 영어 문장 | 뜻·역할·문맥 알아보기 | 문서 리더, 웹 리더 |
| 듣기 카드 | 원본 소리 또는 구간 | 실제 발음에서 문장 알아듣기 | 듣기 루프, 영상 리더 |
| 말하기 카드 | 모국어 뜻 또는 상황 | 영어 문장 직접 만들기 | Life Mining, 말하기 흐름 |

Web Reader의 기본 문장 카드 흐름은 먼저 **읽기 카드**를 만듭니다. 듣기·말하기 카드를 한 번에 모두 만든다고 가정하지 마세요. 각각의 전용 화면에서 필요한 감각에 맞게 만듭니다.

### 7.2 생성 미리보기에서 편집하기

**경로:** 카드 생성 결과 → 편집/본문 편집 → 저장

**쓰는 이유:** AI 설명은 틀리거나 과할 수 있으므로 내 상황에 맞는 카드만 남기기 위해서입니다.

**예시:** `I’m running a little late.`를 “나는 조금 늦게 달리고 있다”로 번역했다면 “나 조금 늦을 것 같아”처럼 상황에 맞게 고칩니다.

**완료 상태:** 미리보기와 저장된 카드에 수정한 문장·뜻·메모가 보입니다.

**막히면:** 저장 후 카드 목록에서 전체 본문 편집 버튼을 찾는 대신, 원본 화면에서 후보를 다시 만들고 저장 전 수정합니다. 현재 카드 관리 화면의 직접 편집 대상은 주로 태그입니다.

**개인정보·비용:** 재생성은 선택한 AI 공급자에 따라 호출·비용이 다시 들 수 있습니다. 단순 텍스트 편집에는 AI 비용이 없습니다.

### 7.3 검색, 필터, 태그

**경로:** 관리 → 카드

카드 덱, 검색어, 태그로 목록을 좁힙니다. 태그 편집에서 `주제:약속`, `의도:늦음알리기`처럼 나중에 찾기 쉬운 분류를 추가할 수 있습니다.

**완료 상태:** `태그를 저장했습니다`가 보이고 필터를 선택했을 때 해당 카드만 남습니다.

**막히면:** 여러 태그는 하나라도 일치하는 방식으로 표시됩니다. 결과가 없다면 태그 선택을 해제하고 검색어를 지웁니다.

**개인정보·비용:** 카드와 태그는 로컬 평문입니다. 동기화 폴더를 연결했다면 카드 JSON이 그 폴더를 관리하는 외부 서비스로 올라갈 수 있습니다.

[![카드 덱 필터와 검색 및 태그 관리](site/assets/app-images/ko/manual-06-card-library.webp)](site/assets/app-images/ko/manual-06-card-library.webp)

*읽기·듣기·말하기 덱을 나누고 검색과 태그로 필요한 카드를 찾습니다.*

### 7.4 기본 샘플과 영구 삭제

새 프로필에는 조작을 이해하기 위한 기본 샘플 카드가 있을 수 있습니다. 샘플 전용의 별도 “숨기기” 버튼이 있다고 가정하지 마세요. 현재는 필요 없는 기본 샘플을 **영구 삭제하면 다시 나타나지 않도록 숨김 상태가 함께 기록**됩니다.

**경로:** 관리 → 카드 → 카드 선택 → 카드 삭제 → 영구 삭제

**완료 상태:** 카드와 그 복습 기록이 사라지고, 삭제한 기본 샘플은 앱을 다시 켜도 재생성되지 않습니다.

**막히면:** 실수로 일반 카드를 삭제했다면 되돌리기 버튼은 없습니다. 최근 `.lembackup`이 있을 때만 복원 방식을 검토합니다.

**개인정보·비용:** 삭제는 로컬 작업이며 비용이 없습니다. 동기화된 다른 장치나 사용자가 따로 복사한 백업까지 자동 삭제하지는 않습니다.

[![샘플 카드 영구 삭제 경고](site/assets/app-images/ko/manual-07-card-delete.webp)](site/assets/app-images/ko/manual-07-card-delete.webp)

*삭제 전에 복습 기록도 사라지고 기본 샘플이 다시 나타나지 않는다는 경고를 확인합니다.*

## 8. 문서 라이브러리, 리더, 최근 문서, 북마크

### 8.1 PDF·HTML 문서 열기

**경로:** 읽고 듣기 · Input → 읽기 → 문서 리더기 → 파일 추가

**쓰는 이유:** 내가 실제로 읽는 자료에서 문장과 문맥을 함께 카드로 남기기 위해서입니다.

1. `.pdf`, `.html`, `.htm` 파일을 고릅니다.
2. 이전·다음 페이지, 확대·축소, 너비 맞춤 또는 페이지 맞춤을 사용합니다.
3. 찾을 문구가 있으면 검색합니다.
4. 텍스트를 선택하고 문장 카드 동작을 엽니다. 기본 단축키는 `Ctrl+Q`이며 설정에서 바꿀 수 있습니다.
5. 선택 문장과 표현을 확인하고 읽기 카드 후보를 저장합니다.

**예시:** 문서에서 `I’m running a little late.`를 선택해 직역, 자연스러운 뜻, 학습 메모를 확인합니다.

**완료 상태:** 카드 저장 안내가 나오고 해당 파일이 최근 문서에 추가됩니다.

**막히면:** 스캔 이미지로만 된 PDF는 텍스트 선택이 안 될 수 있습니다. 파일 자체가 열리지 않으면 암호화·손상 여부와 확장자를 확인합니다. 오류의 전체 로컬 경로는 기본 화면에서 숨겨지고 기술 세부 정보에 접혀야 합니다.

**개인정보·비용:** 원본 파일과 로컬 경로는 이 PC에 남습니다. 클라우드 번역·카드 생성을 선택했을 때만 선택 텍스트가 외부 공급자로 전송되며, 작업 직전 범위와 비용을 다시 확인합니다.

[![로컬 PDF를 여는 문서 리더](site/assets/app-images/ko/manual-08-document-reader.webp)](site/assets/app-images/ko/manual-08-document-reader.webp)

[![선택한 PDF 문장으로 카드를 만드는 미리보기](site/assets/app-images/ko/07-reading-card-preview.webp)](site/assets/app-images/ko/07-reading-card-preview.webp)

*권리 문제가 없는 예제 PDF에서 문장을 선택하고 카드 미리보기까지 확인한 흐름입니다.*

### 8.2 최근 문서 관리

**경로:** 문서 리더기 → 최근 문서

문서 제목 또는 파일명으로 검색하고, 전체·최근·내보내기 결과 필터를 사용합니다. `리더에서 열기`, `파일 위치 열기`, `최근 기록에서 제거`, `최근 기록 비우기`를 사용할 수 있습니다.

**완료 상태:** 선택한 문서가 리더에서 열리거나 최근 목록에서 제거됩니다. 최근 기록을 비워도 내보내기 기록은 유지됩니다.

**막히면:** “최근 기록에서 제거”는 원본 파일을 삭제하는 기능이 아닙니다. 원본이 이동됐다면 기록은 있어도 열 수 없으므로 다시 파일을 추가합니다.

**개인정보·비용:** 최근 목록에는 로컬 파일을 가리키는 정보가 포함될 수 있습니다. 화면 공유 때 파일명을 가리고, 백업에서는 원본 로컬 경로가 제외된다는 점을 기억하세요.

[![최근 문서를 검색하고 다시 여는 문서 라이브러리](site/assets/app-images/ko/manual-09-document-library.webp)](site/assets/app-images/ko/manual-09-document-library.webp)

*최근 기록을 다시 열거나 목록에서 제거합니다. 목록 제거는 원본 파일 삭제가 아닙니다.*

### 8.3 북마크 저장과 다시 열기

**경로:** 문서 리더 상단 → 북마크 / 문서 작업 영역 → 북마크

**쓰는 이유:** 긴 PDF에서 다음에 읽을 페이지나 카드로 만들 문장을 다시 찾기 위해서입니다.

현재 페이지에서 `북마크`를 누르면 프로필별로 저장됩니다. 북마크 목록에서 `저장한 쪽 열기`를 누르면 같은 페이지로 돌아갑니다.

**완료 상태:** 버튼이 선택된 상태가 되고 북마크 목록에 문서 제목과 페이지가 나타납니다.

**막히면:** 원본 파일 위치가 없어졌다면 바로 열 수 없습니다. 파일을 원래 위치로 되돌리거나 문서 라이브러리에서 다시 추가합니다.

**개인정보·비용:** 북마크는 로컬 기록입니다. 백업 후 다른 PC로 복원해도 원본 파일·경로는 따라오지 않으므로 파일을 다시 골라야 할 수 있습니다.

[![저장한 문서 북마크 목록](site/assets/app-images/ko/manual-10-bookmarks.webp)](site/assets/app-images/ko/manual-10-bookmarks.webp)

*문서와 페이지를 확인해 북마크 위치로 돌아가거나 필요 없는 항목을 삭제합니다.*

## 9. 웹 리더

### 9.1 실제 웹페이지에서 읽기 카드 만들기

**경로:** 읽고 듣기 · Input → 읽기 → 웹 리더

**쓰는 이유:** 브라우저와 카드 앱 사이를 오가지 않고, 읽던 자리에서 문장·제목·URL·가까운 문맥을 함께 남기기 위해서입니다.

1. 홈의 목적별 시작점, 내장 연습 글, 또는 주소 입력창을 사용합니다.
2. 학습할 표현을 드래그합니다.
3. `문장카드`를 눌러 읽기 카드 후보를 만듭니다.
4. 자동 저장을 꺼 두었다면 결과를 편집한 뒤 저장합니다.

**완료 상태:** 실제 카드 목록에서 출처가 붙은 읽기 카드를 확인할 수 있습니다.

**막히면:** 사이트가 선택을 막거나 로그인이 필요할 수 있습니다. 사이트 약관을 확인하고, 안 되면 텍스트를 문서로 저장하거나 Life Mining의 직접 추가를 사용합니다.

**개인정보·비용:** 로그인 쿠키는 웹 리더 전용 Electron 세션에 남을 수 있습니다. 설정에서 별도로 삭제할 수 있습니다. 웹페이지를 여는 것과 카드 텍스트를 클라우드 AI로 보내는 것은 별개의 동작입니다.

[![웹 리더에서 실제 문장을 선택하고 문장카드를 누르는 화면](site/assets/app-images/ko/06-web-reader-select.webp)](site/assets/app-images/ko/06-web-reader-select.webp)

*그림 8. 선택만으로 전송되는 것이 아니라 카드 동작을 눌러 후보를 만듭니다.*

### 9.2 번역과 로그인 정보 지우기

**경로:** 웹 리더 상단 → 페이지 번역/선택 번역, 또는 설정 → 고급 → 개인정보·로컬 데이터 삭제 → 웹 리더 로그인 정보 삭제

브라우저 내장 번역은 지원되는 Electron/Chrome 및 언어 조합에서만 동작합니다. Gemini·Google 번역을 골랐다면 별도 클라우드 사전 확인이 나옵니다.

**완료 상태:** 번역 결과가 페이지 또는 선택 영역에 표시되거나, 쿠키 삭제 후 “삭제와 잔존 검증 완료”가 보입니다.

**막히면:** 브라우저 내장 번역이 지원되지 않는다는 안내가 나오면 Local MT, Ollama, Gemini 또는 Google 중 준비된 번역 엔진을 선택합니다. 로그인 삭제 뒤에는 해당 사이트에 다시 로그인해야 합니다.

**개인정보·비용:** 페이지 전체 번역은 생각보다 많은 텍스트를 보낼 수 있습니다. 클라우드 공급자를 쓸 때는 전송 문자 수·호출 상한·예상 비용을 확인하세요.

## 10. 듣기 루프와 영상 리더

### 10.1 듣기 루프로 오늘 문장 만들기

**경로:** 읽고 듣기 · Input → 듣기 → 듣기 루프

**쓰는 이유:** 긴 영상을 끝까지 수동 반복하지 않고, 오늘 들을 짧은 문장 단위로 나눠 실제 소리를 익히기 위해서입니다.

1. 추천 영상을 하나 이상 고르거나 YouTube에서 직접 고릅니다.
2. 오늘 목표 문장 수를 정하고 `루틴 만들기`를 누릅니다.
3. 자막이 없으면 Whisper 전사를 준비합니다. 영상 길이에 따라 시간이 걸릴 수 있습니다.
4. 현재 문장을 반복하고, 잘 안 들리는 표현을 드래그한 뒤 형광펜을 표시합니다.
5. `문장 저장`으로 듣기 카드에 넣습니다.

**예시:** `I’m running a little late.`에서 `running a little`이 뭉쳐 들리지 않으면 그 부분을 표시하고 원본 구간과 함께 저장합니다.

**완료 상태:** `듣기 카드에 저장됨` 안내가 보이고 오늘 들은 문장 수가 증가합니다.

**막히면:** 앱 내부 YouTube 재생이 막혀도 문장 루프와 저장은 계속 사용할 수 있습니다. 영상은 `YouTube에서 보기`로 확인하세요. 루틴에 문장이 부족하면 영상을 더 고르거나 준비된 문장 수로 시작합니다.

**개인정보·비용:** YouTube·RSS 후보와 전사 준비에는 네트워크 또는 로컬 연산이 필요할 수 있습니다. 원본 미디어 파일과 경로는 `.lembackup`에 들어가지 않습니다.

[![듣기 루프에서 자막 문장을 반복하고 듣기 카드로 저장하는 화면](site/assets/app-images/ko/16-listening-loop.webp)](site/assets/app-images/ko/16-listening-loop.webp)

*그림 9. 잘 안 들리는 실제 구간을 반복한 뒤 듣기 카드로 남깁니다.*

### 10.2 영상 리더로 내 영상 공부하기

**경로:** 읽고 듣기 · Input → 듣기 → 영상 리더

**쓰는 이유:** 직접 고른 로컬 영상 또는 YouTube 영상의 자막, 번역, 셰도잉, 카드 저장을 한 작업 공간에서 관리하기 위해서입니다.

1. `파일 열기`로 MP4·MKV·WebM·MOV를 고르거나 YouTube URL을 엽니다.
2. 내장 자막, SRT/VTT, 저장된 자막, 로컬 Whisper 중 준비된 방법을 사용합니다.
3. 문장 루프, 자동 일시정지, 셰도잉, 자막 가리기·흐리기를 조절합니다.
4. 현재 구간과 선택한 표현을 듣기 카드로 저장합니다. 단축키를 쓸 때는 입력창에 포커스가 없는지 확인합니다.

**완료 상태:** 자막 문장 수가 표시되고 재생 위치가 이어보기로 저장되며, 카드에 원본 오디오 또는 영상 구간 상태가 표시됩니다.

**막히면:** 코덱을 직접 재생하지 못하면 앱에서 파일을 다시 열어 재생용 MP4 준비를 시도합니다. Whisper 전사가 끝났는데 문장이 0개면 다른 자막 파일을 불러오거나 전사를 다시 실행합니다. 현재 Whisper 전사에는 중간 취소 버튼이 없으므로 짧은 파일로 먼저 확인하세요. 자막 번역은 실행 중 `중지`할 수 있고, 실패한 작업은 다시 시도할 수 있습니다.

**개인정보·비용:** 로컬 Whisper는 내 PC에서 연산하지만 영상 파일 자체가 백업에 포함되지는 않습니다. 자막 전체를 클라우드 번역하면 원문 자막이 외부로 전송되고 재시도까지 비용이 생길 수 있습니다.

[![로컬 영상과 자막으로 연습하는 영상 리더](site/assets/app-images/ko/manual-11-video-reader.webp)](site/assets/app-images/ko/manual-11-video-reader.webp)

*권리 문제가 없는 로컬 예제 영상에서 자막을 반복하고 듣기 카드로 저장하는 화면입니다.*

## 11. Life Mining

Life Mining은 “공부할 표현을 미리 찾는” 대신, 내가 실제로 말하려다 막힌 내용을 후보로 모으는 기능입니다. 자동 수집은 기본으로 꺼져 있습니다.

### 11.1 직접 후보 추가하기

**경로:** 말하고 쓰기 · Output → Life Mining → 직접추가

1. `내가 한 말`에 “나 조금 늦을 것 같아”를 입력합니다.
2. 필요하면 직전·이후 맥락을 넣습니다.
3. `후보로 저장`을 누릅니다.
4. 후보를 선택하고 `문장생성`을 누릅니다.
5. 결과 미리보기에서 `I’m running a little late.`와 맥락을 확인한 뒤 말하기 카드를 저장합니다.

**완료 상태:** 후보가 처리 완료로 바뀌고 말하기 카드가 복습 대상에 들어갑니다.

**막히면:** 생성에 실패한 항목은 선택된 채 남아 다시 시도할 수 있습니다. 대량 생성 중 취소하면 이미 저장된 카드는 중복 생성하지 않고 미처리 항목을 남깁니다.

**개인정보·비용:** 후보·앞뒤 대화는 로컬 평문입니다. Gemini에는 내가 한 말, 앞뒤 문맥, 학습 언어 설정이 전송됩니다. 원격 Ollama도 외부 전송으로 취급합니다. 비용 확인창의 앱 가드는 실제 청구 차단이 아닙니다.

### 11.2 브라우저 자동 수집 범위 정하기

**경로:** 설정 → 캡처

**쓰는 이유:** 필요한 사이트와 내 메시지만 최소 범위로 수집하기 위해서입니다.

확장 연결 후에도 자동 수집을 직접 켜고, 허용 사이트를 따로 골라야 합니다. 추천 시작값은 `내 말 + 답변 문맥`, 짧은 앞뒤 문맥, 중복 방지, 긴 메시지 제한, 마스킹 사용입니다. Discord, ChatGPT, Claude, YouTube, Reddit, 일반 웹 권한은 필요한 것만 켭니다.

**완료 상태:** 설정에 `브라우저 자동 수집 켜짐`, 연결된 공식 확장 origin, 최근 수집 위치가 표시됩니다.

**막히면:** 토큰 재발급·연결 해제·확장 권한 철회 뒤에는 확장이 자동 재페어링을 시도할 수 있습니다. 계속 실패하면 앱과 확장을 모두 다시 열고 공식 확장 한 개만 연결했는지 확인합니다.

**개인정보·비용:** 허용한 사이트의 내 메시지와 선택한 문맥이 이 PC에 저장됩니다. 전역 키로깅이나 모든 입력 수집은 하지 않습니다. 자동 수집과 AI 카드 생성은 별개이며, 생성 시 다시 전송·비용 확인을 합니다.

## 12. 영작 훈련

**경로:** 말하고 쓰기 · Output → 영작 훈련

**쓰는 이유:** 영어를 보고 이해하는 데서 멈추지 않고 모국어 상황만 보고 문장을 꺼내기 위해서입니다.

1. 내 카드 기반 또는 회화 문장을 고릅니다.
2. 한국어 프롬프트를 보고 영어 답안을 입력합니다.
3. 필요한 경우 힌트를 한 번만 봅니다.
4. `답 보기` 또는 확인 동작으로 내 답안, 표현 체크, 추천 답안을 비교합니다.
5. `다시 쓰기`로 답을 가린 뒤 다시 꺼냅니다.

**예시:** “친구에게 조금 늦을 것 같다고 알려 주세요.”에 `I’m running a little late.`를 입력합니다.

**완료 상태:** 확인 횟수가 늘고 피드백과 추천 답안이 표시됩니다. 일일 영작 미션도 진행됩니다.

**막히면:** 정답 하나를 그대로 맞히는 퀴즈가 아닙니다. 의미가 맞지만 표현이 다르면 추천 답안과 비교해 다음 연습에서 집중할 부분을 정합니다.

**개인정보·비용:** 기본 비교는 로컬입니다. 결과가 자동으로 카드의 SRS 힌트를 바꾼다고 가정하지 마세요. 클라우드 피드백을 추가로 사용하는 동작이 있다면 별도 사전 확인이 떠야 합니다.

[![영작 훈련에서 내 답안, 표현 체크, 추천 답안을 비교하는 화면](site/assets/app-images/ko/12-writing-result.webp)](site/assets/app-images/ko/12-writing-result.webp)

## 13. 캐릭터챗

**경로:** 말하고 쓰기 · Output → 캐릭터챗

**쓰는 이유:** 복습한 문장을 실제 대화의 속도와 맥락에서 다시 쓰기 위해서입니다.

1. 캐릭터와 시나리오를 고릅니다.
2. 모국어로 편하게 대화하는 모드 또는 학습 언어 연습 모드를 고릅니다.
3. 로컬인지 클라우드인지 연결 상태를 확인합니다.
4. 클라우드라면 전송 안내를 읽고 시작합니다.
5. `I’m running a little late.`처럼 복습한 표현을 의도적으로 한 번 사용합니다.
6. 교정 제안은 그대로 복사하기보다 다시 말해 봅니다.

**완료 상태:** 캐릭터 답변, 교정 제안, 가까운 카드 힌트가 보이고 대화가 자동 저장됩니다.

**막히면:** 응답이 멈추면 `중지` 후 다시 보냅니다. 캐릭터가 열리지 않으면 캐릭터팩의 검증 상태와 AI 연결을 구분해 확인합니다.

**개인정보·비용:** 클라우드 공급자는 캐릭터 설정, 최근 대화, 새 메시지, 선택된 카드 힌트를 받을 수 있습니다. 실명, 연락처, 회사 기밀, 의료·금융 정보를 입력하지 마세요. 원격 캐릭터 이미지는 확인 전 자동 로드하지 않으며, 허용은 현재 세션 기준입니다.

### 13.1 캐릭터 만들기·가져오기·내보내기

**경로:** 캐릭터챗 → 캐릭터 관리

이름, 설명, 성격, 시나리오, 첫 메시지, 감정 이미지 세트를 편집할 수 있습니다. JSON 또는 검증 가능한 캐릭터팩을 가져오고, 제작자·소스 URL·라이선스·버전·변경사항을 넣어 팩으로 내보낼 수 있습니다.

**완료 상태:** 가져오기 보안 리포트에 형식, 제작자, 라이선스, SHA-256, 원격 이미지와 경고가 표시되고 안전한 데이터 팩만 등록됩니다.

**막히면:** 레거시 JSON 경고가 뜨면 내용을 먼저 검토하고 새 팩 형식으로 다시 내보냅니다. HTML·JS 실행이 필요한 캐릭터팩은 허용 대상이 아닙니다.

## 14. 복습

**경로:** 복습

**쓰는 이유:** 읽기·듣기·말하기를 같은 방식으로 뭉개지 않고, 각 감각에 맞는 회상을 일정에 따라 반복하기 위해서입니다.

## 15. PlayZone, 공식 게임, UGC

### 15.1 공식 게임 설치하고 실행하기

**경로:** 플레이존 → 공식 게임 선택 → 설치하고 플레이 → 검토하고 다운로드

공식 게임은 앱 설치 파일에 통째로 들어 있지 않습니다. 목록과 검증 정보는 보이지만, 처음 플레이할 때 각 게임의 공개 GitHub Release에서 내려받습니다. 세 게임의 현재 다운로드 합계는 약 166 MiB입니다.

| 공식 게임 | 버전 | 대략적 다운로드 | 소스 | 코드·자산 라이선스 핵심 |
|---|---:|---:|---|---|
| 심연의 무명소환사(Abyss Summoner) | 0.1.2 | 57.0 MiB | [MeowthologySaga/abyss-summoner](https://github.com/MeowthologySaga/abyss-summoner) | 코드 GPL-3.0-only, 공식 자산 별도 조건 |
| 드릴하트 디펜스 | 0.2.0 | 14.2 MiB | [MeowthologySaga/Drillheart_Defense](https://github.com/MeowthologySaga/Drillheart_Defense) | 코드 GPL-3.0-only, 공식 자산 별도 조건 |
| 고양이 오디세이 | 0.1.1 | 94.8 MiB | [MeowthologySaga/Cat_Odyssey](https://github.com/MeowthologySaga/Cat_Odyssey) | 코드 MIT, 이미지·오디오 별도 조건, EP1–20 내레이션·해당 컷신은 비상업 전용 |

앱의 설치 저장 가드는 게임팩 하나당 512 MiB, 전체 1 GiB, 최대 128팩입니다. 게임 저장 데이터는 팩당 5 MiB, 전체 256 MiB로 제한됩니다. 압축 파일과 임시 해제 공간이 추가로 필요할 수 있으므로 이 수치만큼의 빈 공간만 남겨 두면 충분하다는 뜻은 아닙니다.

1. 크기, 제작자, 버전, 소스, 라이선스, SHA-256을 읽습니다.
2. 요청 권한을 확인합니다. 공식 세 게임은 로컬 저장과 선언된 다이아 사용을 요청하고, 네트워크·외부 링크·카드 읽기는 기본 거부입니다.
3. 다운로드를 시작합니다. 중복 클릭하지 말고 진행률을 기다립니다.
4. 압축·경로·해시·manifest·권한 검사가 끝나 `trusted_official`이 된 뒤 `플레이`를 누릅니다.

**완료 상태:** 별도 게임 창이 열리고 실제 캔버스·이미지·저장 상태가 동작합니다.

**막히면:** “게임팩을 불러오는 중”이 오래 지속되면 네트워크, GitHub 접속, 디스크 여유 공간을 확인하고 취소 후 다시 시도합니다. 부분 다운로드를 임의로 풀지 마세요. 보안 상태가 `warning`, `blocked`, `quarantined`이면 실행하지 않습니다.

**개인정보·비용:** 게임 다운로드에는 인터넷 데이터가 들지만 API 요금은 아닙니다. 게임은 CSP와 별도 런타임 안에서 열리며 네트워크는 기본 거부됩니다. 기술 검사는 재미·내용 등급·저작권의 완전한 보증이 아닙니다.

### 15.2 다이아 쓰기

**쓰는 이유:** 학습 보상을 공식 게임의 소환, 부활, 재선택 같은 선택형 기능에 쓰기 위해서입니다.

게임이 요청한 action id, 금액, 사유가 설치된 manifest 선언과 같을 때만 앱 확인창이 열립니다. 예를 들어 드릴하트 디펜스는 즉시 부활 30, 장비 재감정 20, 펫 뽑기 1회 100 다이아를 선언합니다.

**완료 상태:** 확인한 금액만 한 번 차감되고 게임에 성공 응답이 돌아갑니다.

**막히면:** 잔액 부족 또는 선언과 다른 요청은 거절됩니다. 버튼을 여러 번 눌러도 같은 요청의 중복 차감이 일어나서는 안 됩니다. 이상하면 게임을 닫고 다이아 거래 기록과 이슈 양식을 확인합니다.

**개인정보·비용:** 앱 다이아는 현금 결제가 아닙니다. 카드 내용은 공식 게임 권한에 포함되지 않으므로 읽을 수 없어야 합니다.

*게임이 선언한 즉시 부활 사유, 30다이아, 현재 잔액을 확인한 뒤에만 차감할 수 있습니다.*

### 15.3 외부 게임팩과 UGC 참여

**경로:** 플레이존 → 파일 추가 또는 폴더 선택

UGC 게임팩은 먼저 격리된 위치에서 검사합니다. `ready` 또는 `trusted_official`만 실행할 수 있습니다. ZIP 경로 탈출, 심볼릭 링크, 비정상 압축률, CRC 오류, 잘못된 entry, 외부 fetch, 미선언 권한은 차단 대상입니다.

**완료 상태:** 보안 리포트에 제작자, 소스 URL, SPDX 라이선스, 파일 해시, 권한, 경고가 보이고 실행 가능 상태가 명시됩니다.

**막히면:** 차단 코드를 제작자에게 전달하고 고친 새 버전을 받습니다. 경고를 없애려고 파일을 직접 수정하면 해시와 lineage가 달라질 수 있습니다.

**개인정보·비용:** GitHub Releases를 원본으로 권장합니다. Google Drive 같은 링크는 미검증 외부 링크로 봅니다. 공식 Discord는 아직 열리지 않았으므로 초대 링크를 검색하거나 비공식 서버를 공식 커뮤니티로 오해하지 마세요. 직접 만든 팩은 제작자·소스·SPDX 라이선스·권한을 정확히 적어 GitHub Release로 공유하고, 향후 저장소에 커뮤니티 접수 경로가 공지되면 링크를 제출하세요.

## 18. 번역과 TTS

### 18.1 번역 엔진 고르기

**경로:** 설정 → AI · 음성 → 번역 · API

| 엔진 | 데이터 위치 | 키 | 알맞은 용도 |
|---|---|---|---|
| 로컬 번역기(Local MT) | 이 PC | 없음 | PDF 번역, 첫 준비에 모델 다운로드 가능 |
| Ollama LLM | loopback이면 이 PC | 없음 | 로컬 LLM 번역·카드 생성 |
| Gemini | Google | Gemini 키 | 빠른 클라우드 번역·생성 |
| Google 번역 | Google Cloud Translation | 별도 Google 번역 키 | 문서 번역 |
| 브라우저 내장 | 현재 웹 리더 환경 | 없음 | 지원되는 웹페이지·언어 조합 |

**완료 상태:** `연결 성공` 또는 브라우저 번역 가능 상태가 표시됩니다.

**막히면:** Gemini 키와 Google Cloud Translation 키는 서로 다른 입력칸입니다. Local MT 첫 준비는 오래 걸릴 수 있습니다. 브라우저 내장이 현재 조합을 지원하지 않으면 다른 엔진을 고릅니다.

**개인정보·비용:** 클라우드 엔진은 번역 텍스트를 외부로 보냅니다. 키는 URL 쿼리가 아니라 지원되는 인증 방식으로 사용되어야 하며 화면·로그에서 마스킹되어야 합니다.

### 18.2 카드 음성(TTS) 고르기

**경로:** 설정 → AI · 음성 → 카드 음성

TTS는 텍스트를 음성으로 읽는 기능입니다. 현재는 Windows 내장 또는 브라우저 음성을 고르고, 음성 이름과 속도(-10~10)를 조절합니다. Piper는 준비 중이므로 선택할 수 없습니다.

**완료 상태:** 카드의 듣기 버튼에서 선택한 음성과 속도로 문장이 재생됩니다.

**막히면:** Windows에 학습 언어 음성이 설치되어 있는지, 출력 장치가 맞는지 확인합니다. 브라우저 음성은 즉시 재생 방식이라 카드 저장 때 미리 생성할 수 없습니다.

**개인정보·비용:** Windows 내장·브라우저 음성의 실제 처리 정책은 OS/브라우저 환경에 따라 다를 수 있습니다. 앱에 포함된 튜토리얼 샘플 음성은 재생 시 기기 TTS로 만듭니다.

[![번역 엔진과 API 키를 구분하는 설정](site/assets/app-images/ko/manual-18a-translation-settings.webp)](site/assets/app-images/ko/manual-18a-translation-settings.webp)

[![TTS 엔진과 음성 및 속도 설정](site/assets/app-images/ko/manual-18b-tts-settings.webp)](site/assets/app-images/ko/manual-18b-tts-settings.webp)

*카드 생성용 Gemini와 번역용 Google 키를 구분하고, 음성 재생 방식은 별도 TTS 설정에서 고릅니다.*

## 19. 카드 동기화 폴더

**경로:** 설정 → 동기화 · 데이터 → 카드 동기화 / 관리 → 카드 → 동기화 폴더 패널

**쓰는 이유:** 지정한 폴더의 카드 JSON을 다른 장치 또는 백업 폴더와 맞추기 위해서입니다.

1. `폴더 선택`으로 전용 Language Miner 폴더를 고릅니다.
2. 카드 화면에서 의도한 동작을 고릅니다. `업로드`는 현재 로컬 카드 스냅샷을 폴더에 쓰고, `다운로드`는 폴더 카드를 로컬에 가져와 병합하며, `동기화`는 마지막 동기화 스냅샷을 기준으로 양쪽을 병합합니다.
3. 필요하면 앱 시작 시 동기화, 완전 종료 시 동기화를 켭니다.
4. 작업 뒤 결과 메시지에서 업로드·다운로드·충돌 복사본 수를 확인합니다. 같은 카드가 양쪽에서 달라졌다면 앱은 폴더 쪽 내용을 별도의 `충돌` 카드로 보존하므로 카드 목록에서 두 버전을 검토해 하나로 정리합니다.

**완료 상태:** 연결됨, 마지막 카드 파일 변경 시각, 업로드·다운로드·충돌 복사본 수가 포함된 결과 메시지가 보입니다.

**막히면:** 폴더를 지우면 자동 동기화도 꺼집니다. 창의 X는 앱을 트레이로 숨길 수 있으므로 `종료 시 동기화`는 시스템 트레이의 `종료`를 사용할 때 완료되는지 확인합니다.

**개인정보·비용:** OneDrive, Dropbox, Google Drive, iCloud 경로는 카드 JSON을 해당 서비스로 업로드할 수 있습니다. 개인 대화 카드가 있다면 서비스의 공유 설정과 암호화를 먼저 확인하세요. 이 기능은 전체 백업이 아니며 키·쿠키·게임 데이터까지 동기화하지 않습니다.

[![카드 동기화 폴더와 동기화 방향 설정](site/assets/app-images/ko/manual-19-card-sync.webp)](site/assets/app-images/ko/manual-19-card-sync.webp)

*업로드·다운로드·양방향 동기화를 구분하고 외부 동기화 폴더의 업로드 가능성을 확인합니다.*

## 20. AI 연결

새 설치의 기본값은 `AI 미연결`입니다. 카드 저장·복습·기본 화면 탐색은 공용 API 키 없이 사용할 수 있습니다.

[![카드 생성에서 AI 미연결, Ollama, Gemini, ChatGPT 웹 선택지가 보이는 화면](site/assets/app-images/ko/22-ai-options.webp)](site/assets/app-images/ko/22-ai-options.webp)

*그림 18. 필요할 때만 원하는 생성 방식을 연결합니다.*

### 20.1 AI 미연결

**경로:** 설정 → AI · 음성 → 카드 생성 → AI 미연결

**쓰는 이유:** 외부 호출과 비용 없이 앱을 둘러보고 예시 구조를 확인하기 위해서입니다.

**완료 상태:** 설정 요약에 `AI 미연결`이 보이고 외부 요청 없이 예시 카드 구조를 만듭니다.

**막히면:** 실제 문맥 분석 품질이 필요하면 Ollama, Gemini 또는 ChatGPT 웹 수동 방식을 선택합니다.

**개인정보·비용:** 외부 LLM 호출과 API 비용이 없습니다.

### 20.2 Ollama 로컬 모델

**경로:** 설정 → AI · 음성 → 카드 생성 → Ollama

**쓰는 이유:** 지원되는 모델을 내 PC에서 실행해 문장·카드 데이터를 외부 API로 보내지 않기 위해서입니다.

1. Ollama를 별도로 설치합니다.
2. 주소가 `http://127.0.0.1:11434`, `http://localhost:11434`처럼 loopback인지 확인합니다.
3. 설치된 정확한 모델명을 고릅니다. 12B급 모델은 메모리와 속도 요구가 클 수 있습니다.
4. `Ollama 실행 및 연결`을 누릅니다. 앱은 실행을 시도한 뒤 서버와 모델을 따로 확인합니다.

**완료 상태:** `Ollama 연결 성공 · <모델> 모델을 사용할 수 있습니다`가 표시됩니다.

**막히면:** 상태를 구분하세요.

- 실행 파일을 못 찾음: Ollama를 먼저 설치합니다.
- 자동 실행 실패: Ollama 앱을 직접 켠 뒤 다시 시도합니다.
- 서버 연결 실패: 주소와 포트를 확인합니다.
- 서버는 연결됐지만 모델 없음: `ollama list`에 있는 정확한 이름을 고르거나 모델을 내려받습니다.
- 느리거나 메모리 부족: 더 작은 모델로 바꾸고 다른 무거운 프로그램을 닫습니다.

**개인정보·비용:** loopback만 로컬입니다. `192.168.x.x`, 사내 서버, 인터넷 주소는 원격 전송 경고가 나와야 합니다. 로컬 API 요금은 없지만 모델 다운로드 용량, 전기, GPU/CPU 시간이 듭니다.

[![Ollama가 꺼졌거나 모델이 없을 때 다음 행동을 안내하는 연결 상태](site/assets/app-images/ko/23-ollama-not-ready.webp)](site/assets/app-images/ko/23-ollama-not-ready.webp)

*그림 19. “연결 안 됨”을 프로그램·서버·모델 문제로 나눠 확인합니다.*

### 20.3 Gemini 카드 생성

**경로:** 설정 → AI · 음성 → 카드 생성 → Gemini → Gemini API 키

Gemini는 BYOK 방식입니다. 첫 연결 동의는 “이 공급자를 켜도 되는가”를 묻고, 실제 카드·번역 작업 전에는 **별도의 작업 사전 확인**이 다시 나옵니다.

1. Google에서 본인 프로젝트와 키를 준비합니다.
2. 키가 사용할 API만 허용하고 가능한 앱 제한을 겁니다.
3. 공급자 quota와 예산 알림을 설정합니다.
4. 앱에 키를 넣고 첫 연결 동의에서 전송 데이터, 키 저장 위치, 비용 경고를 읽습니다.
5. 실제 작업 전 공급자·모델, 보낼 내용, 예상·최대 호출 수, 비용 범위, 이번 달 앱 누적 추정, 취소 방법을 확인합니다.

**완료 상태:** 모델 연결 성공이 표시되고, 작업을 승인한 경우에만 요청이 시작됩니다.

**막히면:** 키 제한, 활성화한 API, 모델명, quota, 결제 프로젝트, 429·5xx 상태를 확인합니다. 키를 채팅·이슈·스크린샷에 붙이지 마세요.

**개인정보·비용:** OS 암호화를 쓸 수 있으면 Electron `safeStorage`로 키를 암호화해 저장합니다. 쓸 수 없으면 디스크에 평문 저장하지 않고 현재 앱 세션에만 유지합니다. 무료 등급 선택은 0원 보장이 아닙니다. 앱 가드는 Google 청구를 막지 못합니다.

[![Gemini 첫 연결에서 외부 전송, 키 저장, 비용 경고에 동의하는 화면](site/assets/app-images/ko/24-cloud-connection-consent.webp)](site/assets/app-images/ko/24-cloud-connection-consent.webp)

*그림 20. 이 화면은 첫 연결 동의입니다. 실제 작업 직전에는 별도의 사전 확인이 다시 열립니다.*

[![클라우드 작업 전 공급자와 전송 범위 및 비용 확인](site/assets/app-images/ko/manual-20-cloud-job-preflight.webp)](site/assets/app-images/ko/manual-20-cloud-job-preflight.webp)

*키를 노출하지 않고 공급자·모델·전송 내용·호출 상한·비용 범위를 확인합니다.*

### 20.4 Google Cloud Translation

**경로:** 설정 → AI · 음성 → 번역 · API → Google 번역

Google 번역 키는 Gemini 키와 별도입니다. 문서·PDF 텍스트를 Cloud Translation API로 보내는 번역 전용 선택입니다.

**완료 상태:** 번역 엔진 연결 성공 후, 문서 작업 사전 확인에서 Google Cloud Translation과 전송 범위가 표시됩니다.

**막히면:** Cloud Translation API가 프로젝트에 활성화됐는지, 키가 그 API를 허용하는지, quota와 결제가 맞는지 확인합니다.

**개인정보·비용:** 페이지 전체·책 번역은 텍스트와 호출 수가 큽니다. 앱 가드와 별개로 Google Cloud의 quota, 예산 알림, 키 제한을 설정하세요.

### 20.5 ChatGPT 웹 수동 연결

**경로:** 설정 → AI · 음성 → 카드 생성 → ChatGPT 웹

**쓰는 이유:** 별도 API 키 없이 사용 중인 ChatGPT 웹 구독에서 수동으로 생성하기 위해서입니다.

1. 앱이 보여 주는 최종 프롬프트와 포함 문장을 읽습니다.
2. 개인정보를 지운 뒤 `전송 내용을 확인하고 복사`를 누릅니다.
3. 기본 브라우저에서 ChatGPT를 열고 직접 붙여넣습니다.
4. ChatGPT의 전체 답변을 앱 대화상자에 다시 붙여넣습니다.
5. `검증하고 사용`을 눌러 요청 ID와 형식을 검사한 뒤 카드 미리보기로 보냅니다.

**완료 상태:** 검증된 결과가 일반 카드 미리보기로 열립니다.

**막히면:** 답변 일부가 아니라 전체를 붙여넣고, JSON 요청이라면 코드 블록을 포함한 응답을 빠뜨리지 않습니다. 이 방식은 확장·백그라운드 자동 생성에는 쓸 수 없습니다.

**개인정보·비용:** 앱은 ChatGPT 계정, 쿠키, 대화 페이지에 접근하지 않습니다. 복사 순간 클립보드에 내용이 들어가고, 붙여넣는 순간 OpenAI로 전송됩니다. 앱이 OpenAI API를 호출해 별도 사용료를 발생시키는 방식은 아니지만, 사용자가 선택한 ChatGPT 요금제의 구독료·메시지 한도와 OpenAI 데이터 정책은 그대로 적용됩니다.

[![ChatGPT 웹을 수동으로 연결하는 대화상자](site/assets/app-images/ko/manual-21-manual-chatgpt.webp)](site/assets/app-images/ko/manual-21-manual-chatgpt.webp)

*전송할 프롬프트를 직접 검토·복사하고 ChatGPT 응답 전체를 다시 붙여 검증합니다.*

## 21. 전체 백업과 복원

### 21.1 `.lembackup` 만들기

**경로:** 설정 → 동기화 · 데이터 → 전체 백업 · 복원 → 백업 파일 만들기

백업에는 카드, 어휘, 복습, 하이라이트, 프로필, Life Mining, 듣기 전사, 캐릭터 프리셋·대화, 루틴·미션, PlayZone 저장과 로컬 보상 기록이 들어갈 수 있습니다.

다음은 제외됩니다.

- API 키, OAuth 토큰, 로그인 쿠키
- 웹 리더 마지막 세션·탐색 상태와 이 장치의 클라우드 동의 기록
- 로컬 파일 경로, 원본 PDF·오디오·영상, 내보낸 결과 파일
- 내려받아 설치한 PlayZone 게임팩 자체(게임 저장과 로컬 보상 기록은 포함 가능)
- OCR 캡처, 로그, 캐시, 로컬 모델
- TTS·번역 캐시

**완료 상태:** `.lembackup` 파일이 저장되고 성공 파일명이 표시됩니다.

**막히면:** 저장 위치의 여유 공간과 쓰기 권한을 확인합니다. 현재 앱이 열 수 있는 단일 백업 파일의 상한은 64 MiB입니다. 실패해도 원본 학습 데이터가 바뀌지 않는지 상태 안내를 확인합니다.

**개인정보·비용:** API 키는 빠져도 카드·대화·복습 기록은 백업 안에 평문으로 들어갈 수 있습니다. 암호화된 개인 저장소에 보관하고 공유하지 마세요.

[![전체 백업에서 포함·제외 항목과 백업 파일 만들기 버튼이 보이는 화면](site/assets/app-images/ko/25-backup-create.webp)](site/assets/app-images/ko/25-backup-create.webp)

*그림 21. 학습 기록을 묶되 비밀 키와 원본 파일은 제외합니다.*

### 21.2 복원 미리보기와 세 방식

**경로:** 같은 화면 → 백업 파일 열기

앱은 형식, 크기, 체크섬, 버전, 손상, 프로필 충돌을 확인하고 미리보기를 엽니다. 미리보기 세션은 30분 뒤 만료되므로 실제 복원을 늦게 시작했다면 파일을 다시 여세요.

| 방식 | 현재 데이터 | 백업 데이터 | 알맞은 경우 |
|---|---|---|---|
| 새 프로필로 복원 | 유지 | 새 ID로 추가 | 먼저 안전하게 살펴볼 때 |
| 겹치지 않는 데이터 합치기 | 현재 값 우선 | 없는 항목만 추가 | 두 장치 기록을 합칠 때 |
| 현재 데이터 교체 | 자동 안전 백업 후 교체 | 백업 기준 | 이 장치를 백업 시점으로 되돌릴 때 |

## 23. 브라우저 확장 수동 설치와 대기열 삭제

`v0.1.0-beta.1`에서는 Chrome Web Store가 아니라 선택형 수동 설치 파일을 제공합니다.

### 23.1 정확한 설치 순서

1. 앱과 같은 공식 Release에서 `Language-Miner-Extension-<version>.zip`과 `SHA256SUMS.txt`를 받습니다.
2. ZIP의 SHA-256이 **같은 파일명**의 체크섬과 일치하는지 확인합니다.
3. 빈 폴더에 ZIP을 압축 해제합니다.
4. Chrome에서 `chrome://extensions`를 엽니다.
5. `개발자 모드`를 켜고 `압축해제된 확장 프로그램을 로드합니다`를 누릅니다.
6. **`manifest.json`이 바로 보이는 폴더**를 선택합니다. ZIP 파일이나 그 위 부모 폴더를 고르면 안 됩니다.
7. 앱 설정 → 캡처에서 공식 확장 연결 상태와 허용 사이트를 확인합니다.

**완료 상태:** 확장 카드에 Language Miner Web Capture가 보이고 앱 설정에 연결된 origin이 표시됩니다.

**막히면:** manifest를 찾지 못한다면 한 단계 안쪽 폴더를 선택합니다. 앱이 꺼져 있을 때 보낸 항목은 대기열에 남았다가 재연결 후 전송됩니다.

**개인정보·비용:** 일반 웹에서 선택 카드 기능을 쓰기 위해 넓은 사이트 접근 권한을 요청할 수 있습니다. 텍스트를 드래그하는 것만으로 보내지 않고 카드 동작을 눌렀을 때 선택 표현, 원문 문장, 제목, URL, 가까운 문맥을 로컬 앱으로 보냅니다. 원치 않으면 확장을 설치하지 않아도 데스크톱 앱의 나머지 기능을 사용할 수 있습니다.

### 23.2 오프라인 대기열 확인·삭제

**경로:** Chrome → 확장 프로그램 관리 → Language Miner Web Capture → 세부정보 → 확장 프로그램 옵션

대기 항목은 7일 뒤 만료됩니다. 한 큐는 최대 200개·2 MiB, 항목 하나는 최대 128 KiB, 모든 대기 큐 합계는 5 MiB로 제한됩니다.

1. `새로고침`으로 개수와 용량을 확인합니다.
2. `대기 항목 모두 삭제`를 누릅니다.
3. 확인 대화상자에서 삭제합니다.
4. 앱이 켜져 있으면 삭제 확인 응답까지 확인합니다.

**완료 상태:** 대기 0개로 표시되고 앱도 삭제 확인을 받습니다.

**막히면:** 앱 확인만 대기 중이면 앱을 열어 연결 상태를 회복합니다. 앱의 캡처 설정 화면에는 이 삭제 버튼이 없다는 점을 기억하세요.

**개인정보·비용:** 큐에는 아직 앱으로 보내지 못한 캡처 내용이 Chrome 로컬 저장소에 잠시 남을 수 있습니다. 공용 PC에서는 직접 비운 뒤 확장을 제거합니다.

## 24. 자주 생기는 문제

| 증상 | 먼저 확인 | 다음 행동 | 보내면 안 되는 것 |
|---|---|---|---|
| 시작할 때 2초 정도 검은 화면 | 앱이 로컬 DB와 화면을 준비 중인지 | 계속되거나 길어지면 버전·PC 사양·발생 시간을 이슈로 기록 | 사용자 폴더 전체 경로 |
| 첫 실행 팝업이 매번 뜸 | 설치판/포터블을 번갈아 실행했는지, 완전 종료 후에도 반복되는지 | 온보딩 완료 상태가 저장되지 않는 버그로 신고 | 개인 프로필 이름 |
| 설정을 누를 때 화면 전체가 새로고침됨 | 자동 저장 안내와 실제 창 재로딩을 구분 | 재현되는 정확한 설정 항목과 짧은 화면 녹화 제공 | API 키가 보이는 녹화 |
| Ollama 연결 실패 | 프로그램, 서버 주소, 모델 존재를 각각 확인 | `Ollama 실행 및 연결`, 직접 실행, 정확한 모델 선택 순으로 재시도 | 원격 서버 인증 정보 |
| Gemini/Google 401·403 | 키 종류, 활성 API, 키 제한 | 공급자 콘솔에서 제한과 프로젝트 확인 | 키 원문 |
| 429·timeout·5xx | quota, 동시 작업, 공급자 상태 | 작업 취소 후 기다리고 작은 범위로 재시도 | 카드의 민감한 원문 |
| 비용 추정과 청구가 다름 | 앱 값은 로컬 추정치인지 | 공급자 콘솔의 실제 사용량·청구를 기준으로 판단 | 결제 계정 개인정보 |
| 문서·미디어가 복원 후 안 열림 | 원본 파일·경로는 백업 제외 | 파일을 다시 선택하고 북마크를 재연결 | 전체 로컬 경로 스크린샷 |
| 게임 다운로드 무한 대기 | GitHub 접속, 디스크 공간, 진행 상태 | 취소 후 다시 다운로드하고 보안 리포트 확인 | 사용자 데이터 폴더 |
| UGC가 실행 안 됨 | `blocked`/`warning`/`quarantined`와 코드 | 제작자에게 코드와 버전을 전달해 새 팩 요청 | 팩 안의 개인 파일 |
| 확장 후보가 안 들어옴 | 앱 실행, pairing, 사이트 허용, 큐 | 확장 옵션 새로고침 후 재페어링 | 대화 원문 전체 |
| SmartScreen 경고 | 공식 Release와 SHA-256 | 둘 다 검증할 수 없으면 실행하지 않음 | 보안 경고를 무시한 우회법 |

일반 버그는 공개 이슈 양식을 사용하되 로그와 화면을 먼저 검토해 이름, 이메일, API 키, 전체 경로, 원문 대화가 없는지 확인하세요. 키 노출·개인정보 유출·보안 취약점은 공개 이슈에 올리지 말고 [비공개 보안 신고 절차](../SECURITY.md)를 사용합니다.

## 25. 처음 일주일 추천 사용법

1. 하루에 읽기 카드 3–5장만 실제 문장에서 만듭니다.
2. 듣기 루프에서 짧은 문장 10–30개를 듣고 정말 안 들리는 구간만 카드로 남깁니다.
3. Life Mining에는 내가 실제로 말하고 싶었던 것 1–3개를 직접 추가합니다.
4. 복습 예정 카드를 먼저 끝냅니다.
5. 그날 카드 중 한 문장을 영작과 캐릭터챗에서 다시 씁니다.
6. 주 1회 `.lembackup`을 만들고 안전한 개인 저장소에 보관합니다.

카드 수를 빠르게 늘리는 것보다, `I’m running a little late.`처럼 한 문장을 **읽고 → 듣고 → 떠올리고 → 직접 쓰는 경험**을 반복하는 편이 Language Miner의 장점을 더 잘 살립니다.
