## 한눈에 보기

Language Miner는 계정이나 중앙 서버 없이 동작하는 로컬 우선 데스크톱 앱입니다.

- 개발자가 운영하는 광고, 분석, 텔레메트리 서버가 없습니다.
- 새 설치에서는 AI가 연결되지 않습니다.
- 카드, 복습, 대화와 설정은 기본적으로 이 PC에 저장됩니다.
- 사용자가 클라우드 기능을 켜면 선택한 내용이 해당 공급자에게 직접 전송됩니다.
- 앱의 비용 가드는 로컬 추정치이며 공급자의 실제 청구를 차단하지 않습니다.
- API 키만 OS 암호화 저장소를 사용합니다. 학습 데이터와 대화는 로컬 평문으로 저장될 수 있습니다.

이 문서는 앱이 직접 처리하는 데이터 흐름을 설명합니다. 사용자가 연결하는 Google, Gemini, Ollama 서버, 웹사이트, GitHub, Google Drive와 Discord에는 각 서비스의 약관과 개인정보처리방침이 별도로 적용됩니다.

## API 키와 연결 자격 증명

Gemini와 Google API 키는 Electron `safeStorage`를 통해 OS 암호화가 가능할 때만 지속 저장합니다.

- OS 암호화를 사용할 수 없으면 키를 평문 파일이나 일반 설정에 저장하지 않습니다.
- 이 경우 키는 현재 앱 세션에서만 사용하며 앱을 다시 열면 다시 입력해야 합니다.
- 키는 요청 URL 쿼리가 아니라 해당 API가 지원하는 인증 헤더로 보냅니다.
- 화면, 오류, 로그와 QA 보고서에는 키를 마스킹합니다.
- `.lembackup`에 API 키, OAuth 토큰과 쿠키를 넣지 않습니다.

Google Drive 카드 동기화를 사용하면 OAuth 토큰이 로컬 자격 증명 파일에 저장될 수 있고, 카드 스냅샷이 사용자의 Google Drive `appDataFolder`로 전송됩니다. 동기화는 사용자가 연결하고 실행한 경우에만 사용합니다.

## 외부 AI와 번역

### AI 미연결

새 설치의 기본 상태입니다. 동의 전에는 Gemini·Google AI 요청을 보내지 않습니다.

### Ollama

`localhost`, `127.0.0.1`, `::1`처럼 현재 기기를 가리키는 loopback 주소만 “로컬”로 표시합니다. LAN 주소나 인터넷 URL은 원격 연결이며 입력 텍스트가 해당 서버 운영자에게 전달될 수 있습니다.

### Gemini 카드 생성·대화

선택한 기능에 따라 다음 항목이 전송될 수 있습니다.

- 카드로 만들 문장과 앞뒤 맥락;
- 학습 언어와 카드 형식 지시;
- 캐릭터 설정과 상황;
- 최근 대화와 새 메시지;
- 관련 카드 힌트.

### ChatGPT 웹 수동 브리지

ChatGPT 구독은 API 사용권으로 전환되지 않습니다. 이 모드는 API 키나 ChatGPT 로그인 쿠키를 사용하지 않습니다. 앱은 전송할 전체 프롬프트를 먼저 보여주고, 사용자가 동의 버튼으로 클립보드에 복사한 뒤 기본 브라우저의 ChatGPT에 직접 붙여넣도록 합니다. 앱은 클립보드를 자동으로 읽거나 ChatGPT 응답을 자동 수집하지 않습니다.

붙여넣은 카드 응답은 요청 ID, 크기, JSON 구조와 허용 필드를 로컬에서 검사합니다. 카드 ID, 프로필, 복습 상태와 저장 시각은 ChatGPT 응답에서 받지 않고 앱이 관리합니다.

### Google Cloud Translation

번역을 요청한 문장 또는 문서 구간, 원문·목표 언어 정보가 Google API로 전송될 수 있습니다.

클라우드 작업 전 화면에서 공급자, 모델, 전송 내용, 예상 호출 수와 비용 범위, 취소 방법을 확인합니다. 동의는 버전과 함께 로컬에 기록합니다.

민감한 개인정보, 비밀번호, API 키, 업무상 비밀, 의료·금융 기록을 번역이나 대화 입력에 넣지 마세요.

## 비용과 사용량

앱에 표시되는 비용과 한도는 이 기기에서 계산한 추정치입니다.

- 실제 토큰 계산, 과금 단위, 무료 할당량, 환율과 세금이 다를 수 있습니다.
- 재시도와 fallback 호출도 사용량을 만들 수 있습니다.
- 앱의 “예상 비용 가드”는 새 작업을 멈추는 보조 장치이며 Google 결제 계정의 청구를 차단하지 않습니다.
- Google Cloud Console에서 키의 API·앱 제한, quota와 예산 알림을 따로 설정하세요.
- 공급자 콘솔의 실제 사용량과 청구 내역을 최종 기준으로 보세요.

## 웹 리더와 원격 콘텐츠

웹 리더에서 로그인하면 사이트 쿠키와 세션 데이터가 로컬 Electron 세션에 저장될 수 있습니다. 개발자 서버로 보내지는 않지만, 접속한 사이트와 해당 사이트가 포함한 제3자에게는 일반 브라우징과 같은 네트워크 요청이 발생합니다.

원격 캐릭터 이미지는 자동으로 불러오지 않습니다. 사용자가 확인해 로드하면 이미지 호스트는 IP 주소, 시각, 일반 요청 헤더를 볼 수 있습니다.

## 브라우저 확장과 Life Mining

확장은 수동 설치형 선택 기능이며 자동 수집은 기본적으로 꺼져 있습니다.

문장 카드 선택 도구는 일반 `http`/`https` 페이지에서 동작하므로 Chrome은 “방문한 웹사이트의 데이터 읽기 및 변경”과 같은 넓은 권한 경고를 표시할 수 있습니다. 텍스트를 선택하는 것만으로는 전송하지 않습니다. 사용자가 문장 카드 버튼을 누른 뒤에만 선택한 표현, 출처 문장, 페이지 제목·URL과 가까운 문맥을 loopback 앱으로 보냅니다. Life Mining의 자동 메시지 수집은 별도의 허용 사이트 설정을 따르며 기본값은 꺼짐입니다.

사용자가 선택하는 항목:

- 허용 사이트;
- 본인 메시지만 수집할지 여부;
- 앞뒤 문맥 포함 여부와 개수;
- 최대 메시지 길이와 긴 메시지 처리;
- 마스킹 규칙.

필터와 마스킹은 앱에 저장하기 전에 적용합니다. 앱이 오프라인이면 항목이 확장 저장소의 제한된 대기 큐에 남을 수 있으며, TTL·건수·용량 상한을 적용합니다. 설정에서 대기 항목을 모두 삭제할 수 있습니다.

## 백업

`.lembackup`에 포함될 수 있는 내용:

- 카드, 어휘, 복습, 하이라이트와 프로필;
- Life Mining 기록과 듣기 전사;
- 캐릭터 프리셋·대화;
- 루틴·미션, PlayZone 저장과 로컬 보상 기록.

제외되는 내용:

- API 키, OAuth 토큰, 쿠키;
- 웹 리더 세션·탐색 상태와 이 기기의 클라우드 외부 전송 동의 기록;
- 로컬 파일 경로, 원본 PDF·오디오·영상;
- OCR 캡처, 로그, 캐시와 모델;
- 별도로 내보낸 파일.

백업은 암호화 파일 형식이 아닙니다. 개인 대화와 학습 기록이 들어갈 수 있으므로 안전한 위치에 보관하세요. OneDrive 같은 동기화 폴더에 저장하면 해당 서비스로 업로드될 수 있습니다.

## 보관과 삭제

로컬 데이터는 사용자가 삭제하거나 Windows 사용자 데이터를 제거할 때까지 남을 수 있습니다. 설정에서 다음 작업을 제공합니다.

- 웹 리더 로그인 쿠키 삭제;
- 확장 대기 큐 삭제;
- API 키 삭제;
- OCR·TTS·번역 캐시 삭제;
- 전체 데이터 삭제.

전체 삭제는 SQLite와 백업 DB, 일반 설정, 보안 설정, 쿠키, 확장 큐와 캐시를 대상으로 합니다. 사용자가 별도 폴더에 복사한 `.lembackup`, PDF와 미디어, Google Drive의 동기화 사본은 자동으로 지우지 않습니다.

앱 제거만으로 사용자 데이터가 모두 삭제된다고 가정하지 마세요. PC를 양도하기 전에는 앱의 전체 삭제를 먼저 실행하고 별도 백업·내보내기 파일도 직접 확인하세요.

## UGC와 커뮤니티 링크

앱은 UGC에 기술적 보안 검사를 적용하지만, 외부에서 받은 모든 콘텐츠의 정확성, 합법성, 저작권, 내용 등급을 보증하지 않습니다. 공식 Discord는 아직 열리지 않았으며, 향후 열리면 일반 이용가 정책을 적용합니다. 외부 링크의 콘텐츠는 공식 추천으로 간주하지 마세요.

UGC 제작자는 팩 안에 API 키, 개인 대화, 로컬 경로, 추적 픽셀과 원격 실행 코드를 넣어서는 안 됩니다. 자세한 내용은 [UGC 정책](ugc-policy.ko.md)을 참고하세요.

## 문의와 변경

일반적인 문서 오류는 GitHub 문서 이슈로 알려 주세요. 취약점, 키 노출 또는 개인정보 유출 가능성은 [GitHub 비공개 보안 신고](../SECURITY.md)를 사용하세요. 실제 키나 개인 데이터를 신고 내용에 넣지 마세요.

데이터 흐름이 바뀌면 이 문서의 수정 날짜와 대상 버전을 갱신합니다. 릴리스마다 포함된 문서를 해당 버전의 기준으로 사용하세요.
