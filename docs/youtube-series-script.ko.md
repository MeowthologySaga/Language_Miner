# Language Miner 완전 사용법 — 5편 영상 시리즈 대본

이 대본은 [완전 사용 설명서](complete-user-manual.ko.md)와 같은 실제 예문·화면 순서를 씁니다. 목표는 짧은 홍보 영상이 아니라, 처음 설치한 사람이 화면을 멈춰 가며 그대로 따라 할 수 있는 사용법 시리즈입니다.

현재 `v0.1.0-beta.1` 공개 배포를 준비하고 있으므로 “이미 출시됐다”고 말하지 않습니다. 실제 Release가 게시된 뒤에만 다운로드 주소, 버전, 파일 크기, Discord 운영 상태를 최종 대사와 자막에 반영합니다.

## 촬영 공통 규칙

- 기본 캔버스는 16:9, 1920×1080입니다. 앱은 1240×820 이상으로 띄우고 중요한 영역을 150–175%까지 후반 확대합니다.
- 마우스 클릭 뒤 0.4초, 화면 전환 뒤 1초 멈춰 시청자가 상태 변화를 볼 시간을 줍니다.
- 실제 UI만 촬영합니다. 문서에 연결된 QA 캡처를 기준으로 동선을 맞추고, 배포 후에만 존재하는 화면은 공개판에서 직접 추가 촬영합니다.
- API 키, 이메일, Windows 사용자명, 전체 로컬 경로, 브라우저 계정 사진, 개인 대화, 결제 프로젝트, 캐릭터 대화 기록은 촬영 전 예제 값으로 바꾸거나 후반 블러 처리합니다.
- 비밀번호·API 키 입력 장면은 실제 타이핑을 찍지 않습니다. 빈 입력칸과 “등록됨” 상태만 서로 다른 테이크로 촬영합니다.
- 클릭 가능한 원본 스크린샷을 글에 넣을 때는 WebP 원본을 유지하고, 움직임이 꼭 필요할 때만 GIF를 씁니다. GIF는 용량을 확인하고 긴 영상 대신 짧은 루프만 사용합니다.
- `I’m running a little late.`를 1편에서 발견하고, 2편에서 듣고, 3편에서 직접 쓰고 대화하며, 5편에서 백업·삭제 예시로 다시 확인합니다.

# 1편. 처음 10분: 문장 하나가 카드·복습·영작이 되는 과정

목표 길이: 16분. 이 편만 본 시청자도 AI 키 없이 실제 읽기 카드 하나와 첫 복습까지 끝낼 수 있어야 합니다.

## 장면 1 — 왜 단어를 외웠는데 말이 안 나올까 (00:00–01:15)

- **화면:** 검은 배경에 `late = 늦은`이 먼저 나오고, 실제 앱 학습 루프 GIF로 전환합니다.
- **클릭·입력:** 없음. `I’m running a little late.`를 단어 → 문장 → 상황 순서로 자막 애니메이션합니다.
- **내레이션:** “late라는 단어를 외웠는데도 약속에 늦는 순간 영어가 바로 나오지 않을 수 있습니다. 단어 뜻만 기억했지, 어떤 문장으로 어떤 상황에서 쓰는지는 함께 연습하지 않았기 때문입니다. Language Miner는 읽거나 듣다가 실제 문장을 발견하고, 카드로 남기고, 잊기 전에 복습하고, 영작과 대화에서 다시 꺼내는 앱입니다. 오늘은 ‘I’m running a little late.’ 한 문장으로 이 전체 루프를 시작해 보겠습니다.”
- **화면 자막:** `단어 암기 → 문장 발견 → 카드 → 복습 → 다시 사용`
- **캡처 확인:** GIF에 마우스나 디버그 오버레이가 없고 첫·마지막 프레임이 자연스럽게 이어지는지 확인합니다.
- **개인정보 가림:** 없음.
- **B-roll 매핑:** `docs/site/assets/app-images/ko/00-learning-loop.gif`.

## 장면 2 — 설치 파일과 첫 실행 언어 (01:15–03:20)

- **화면:** 실제 공개 후 GitHub Release Assets, SHA-256 확인 화면, 첫 실행 언어 팝업 순서입니다.
- **클릭·입력:** 설치판과 포터블판을 가리키고, 첫 실행에서 `한국어 / 한국어 / English`를 선택한 뒤 다음을 누릅니다. AI 단계에서는 `앱 둘러보기`를 고릅니다.
- **내레이션:** “배포 후에는 반드시 같은 공식 Release에서 설치판 또는 포터블판과 SHA256SUMS를 받습니다. 설치판은 NSIS라는 일반 Windows 설치 형식이고, 포터블은 설치 과정이 없는 실행 파일입니다. 포터블이라고 해서 학습 데이터가 전부 실행 파일 옆에만 남는 것은 아닙니다. 첫 실행에서는 앱 언어, 설명의 기준이 되는 모국어, 실제로 공부할 언어를 따로 고릅니다. AI는 아직 연결하지 않아도 됩니다. 새 설치의 기본값은 AI 미연결입니다.”
- **화면 자막:** `앱 언어 ≠ 모국어 ≠ 학습 언어` / `첫 시작은 AI 미연결로 가능`
- **캡처 확인:** Release가 실제 게시되기 전에는 Assets 장면을 녹화하지 않습니다. 첫 실행 팝업을 닫은 뒤 재실행해 반복되지 않는 테이크를 확인합니다.
- **개인정보 가림:** GitHub 프로필, 다운로드 폴더, Windows 사용자명, 브라우저 북마크 바.
- **B-roll 매핑:** `docs/site/assets/app-images/ko/01-onboarding-language.webp`.
- **공개판 추가 촬영:** 실제 Release가 게시된 뒤 설치판·포터블판·체크섬·SBOM이 함께 보이는 Assets 화면과 SHA-256 비교 과정을 촬영합니다.

## 장면 3 — 튜토리얼은 연습 공간, 실제 카드는 웹 리더에서 (03:20–05:10)

- **화면:** 관리 → 튜토리얼의 샌드박스 배너를 5초 보여 준 뒤 나가기, 읽고 듣기 · Input → 웹 리더 홈으로 이동합니다.
- **클릭·입력:** 튜토리얼에서 읽기 카드 연습을 한 번 누르고 실제 카드 목록이 바뀌지 않는 것을 짧게 보여 줍니다. 이후 실제 웹 리더의 `Language Miner 연습 글`을 엽니다.
- **내레이션:** “먼저 중요한 구분이 있습니다. 관리 메뉴의 튜토리얼은 실제 화면을 닮은 연습용 복제 공간입니다. 여기서 만든 것은 튜토리얼 진행만 기록하고 실제 카드 목록은 바꾸지 않습니다. 실제 카드를 만들 때는 튜토리얼을 나와 웹 리더를 엽니다. 연습 글의 주소는 lem-practice://reader/getting-started이고 앱에 들어 있는 글이라 외부 사이트에 접속하지 않습니다.”
- **화면 자막:** `튜토리얼 = 샌드박스` / `실제 연습 글 = lem-practice://reader/getting-started`
- **캡처 확인:** 튜토리얼 배너와 실제 앱 네비게이션의 차이가 한눈에 보이도록 같은 배율로 촬영합니다.
- **개인정보 가림:** 없음.
- **B-roll 매핑:** `docs/site/assets/app-images/ko/manual-01-tutorial-sandbox.webp`, `docs/site/assets/app-images/ko/06-web-reader-select.webp`.

## 장면 4 — 표현 선택, 읽기 카드 편집, 저장 (05:10–08:00)

- **화면:** 내장 연습 글 본문, 선택 팝오버, 읽기 카드 미리보기입니다.
- **클릭·입력:** `running a little late`를 드래그 → `문장카드` → `본문 편집` → 자연스러운 뜻을 확인 → 저장합니다.
- **내레이션:** “표현만 떼어 외우지 않고, 이 표현이 들어 있는 문장 전체를 가져옵니다. 드래그만 했다고 외부로 전송되는 것은 아닙니다. 문장카드 버튼을 눌러 후보를 만들 때 현재 카드 엔진이 동작합니다. 웹 리더의 기본 흐름은 읽기 카드입니다. 듣기 카드와 말하기 카드를 동시에 만들어 주는 화면이 아닙니다. AI가 만든 직역, 자연스러운 뜻, 학습 메모는 저장 전에 직접 고칠 수 있습니다. ‘달리고 있다’처럼 문맥과 다른 번역이면 ‘나 조금 늦을 것 같아’로 바로잡은 뒤 저장합니다.”
- **화면 자막:** `Web Reader → 읽기 카드` / `AI 결과는 저장 전에 직접 편집`
- **캡처 확인:** 선택 범위가 정확하고 미리보기에서 영어 문장, 뜻, 편집, 저장 버튼이 동시에 보이는지 확인합니다.
- **개인정보 가림:** 주소창은 내장 연습 주소만 보이게 합니다.
- **B-roll 매핑:** `docs/site/assets/app-images/ko/06-web-reader-select.webp`, `docs/site/assets/app-images/ko/07-reading-card-preview.webp`.

## 장면 5 — 세 카드와 샘플 카드 정리 (08:00–10:10)

- **화면:** 관리 → 카드의 세 덱 필터, 태그 편집, 기본 샘플 삭제 확인창입니다.
- **클릭·입력:** 읽기·듣기·말하기 필터를 차례로 누르고 `주제:약속`, `의도:늦음알리기` 태그를 저장합니다. 필요 없는 기본 샘플을 선택해 삭제 확인까지만 진행한 뒤 별도 테스트 프로필에서 영구 삭제 결과를 보여 줍니다.
- **내레이션:** “읽기 카드는 영어 문장을 보고 뜻과 문맥을 떠올립니다. 듣기 카드는 실제 소리에서 문장을 알아듣고, 말하기 카드는 한국어 뜻이나 상황에서 영어를 꺼냅니다. 저장된 카드 화면에서는 현재 태그 편집과 삭제를 중심으로 관리하므로 본문 수정은 생성 미리보기에서 끝내는 것이 좋습니다. 새 프로필의 기본 샘플에는 별도 일반 숨김 버튼이 있는 것이 아닙니다. 기본 샘플을 영구 삭제하면 복습 기록도 사라지고, 그 샘플이 다시 나타나지 않도록 숨김 상태가 함께 기록됩니다.”
- **화면 자막:** `읽기 / 듣기 / 말하기` / `기본 샘플 영구 삭제 = 다시 나타나지 않음`
- **캡처 확인:** 삭제는 반드시 QA 프로필에서만 실제 실행합니다. 일반 카드 복습 기록도 함께 사라진다는 경고가 읽히는지 확인합니다.
- **개인정보 가림:** 개인 카드 제목·출처·태그는 예제 데이터로 바꿉니다.
- **B-roll 매핑:** `docs/site/assets/app-images/ko/manual-06-card-library.webp`, `docs/site/assets/app-images/ko/manual-07-card-delete.webp`.

## 장면 6 — 첫 복습과 영작 (10:10–12:50)

- **화면:** 복습 앞면, 답면, 영작 결과 순서입니다.
- **클릭·입력:** 답 보기 전에 2초 멈춤 → 답 보기 → `좋음` → 영작 훈련에서 `I’m running a little late.` 입력 → 답 보기 → 다시 쓰기.
- **내레이션:** “복습에서는 답을 보는 속도보다 보기 전에 떠올리는 시간이 중요합니다. 어렵게 떠올렸다면 다시 또는 어려움, 바로 떠올렸다면 좋음 또는 쉬움을 누릅니다. 이것은 점수가 아니라 다음 복습 간격을 정하는 신호입니다. 이제 한국어 상황만 보고 같은 문장을 직접 씁니다. 영작 결과는 내 답과 추천 표현을 비교해 다음 연습에서 집중할 부분을 고르는 자료입니다. 결과가 카드의 SRS 힌트를 몰래 자동 수정하는 것은 아닙니다.”
- **화면 자막:** `답 보기 전 회상` / `평가 = 다음 간격` / `영작 결과 = 다음 연습의 초점`
- **캡처 확인:** 평가 버튼 네 개와 영작의 내 답안·표현 체크·추천 답안이 읽히는지 확인합니다.
- **개인정보 가림:** 없음. 고정 예문만 사용합니다.
- **B-roll 매핑:** `docs/site/assets/app-images/ko/09-review-front.webp`, `10-review-answer.webp`, `12-writing-result.webp`.

## 장면 7 — 오늘 루틴, 미션, 다이아 (12:50–15:00)

- **화면:** 오늘 작업대, 진행 중 루틴, 미션 패널입니다.
- **클릭·입력:** 오늘 루틴 시작 → 복습 → 듣기 → 영작 → 보상 단계 미리보기. 완료된 테스트 미션의 `받기`를 누릅니다.
- **내레이션:** “오늘 화면은 복습, 새 카드 후보, 듣기 루프를 한곳에 모읍니다. 무엇부터 할지 모르겠다면 복습, 듣기, 영작, 보상 정리 순서의 루틴을 시작하세요. 미션은 읽기 카드 5장 15다이아, 듣기 문장 30개 20다이아, 영작 3회 15다이아, Life Mining 카드 5장 25다이아, 세 덱 복습 완료가 각각 15다이아입니다. 기본 보상을 모두 받은 뒤 오늘 보너스 30다이아가 열립니다. 진행만 채우고 끝내지 말고 받기 버튼을 눌러야 합니다. 다이아는 현금이 아니라 이 기기의 로컬 학습 보상입니다.”
- **화면 자막:** `복습 → 듣기 → 영작 → 보상` / `완료 후 반드시 받기` / `다이아 = 현금 아님`
- **캡처 확인:** 테스트 데이터로 수령 전·후 잔액이 정확히 한 번만 변하는지 촬영합니다.
- **개인정보 가림:** 공부 잔디 날짜 외 개인 활동 설명은 보이지 않게 합니다.
- **B-roll 매핑:** `docs/site/assets/app-images/ko/manual-02-today-hub.webp`, `docs/site/assets/app-images/ko/manual-03-today-routine.webp`, `docs/site/assets/app-images/ko/manual-04-daily-missions.webp`.

## 장면 8 — 프로필과 다음 편 예고 (15:00–16:00)

- **화면:** 상단 프로필 전환과 설정 → 일반 → 프로필 관리입니다.
- **클릭·입력:** 새 테스트 프로필 생성, 복제, 전환을 보여 주고 삭제 확인창은 열기만 합니다.

- **캡처 확인:** 공개 Release 전에는 로컬 QA ZIP으로 화면 흐름만 촬영하고 다운로드 장면은 나중에 교체합니다.

- **클릭·입력:** 키 삭제, 웹 로그인 삭제, 캐시 삭제를 테스트 프로필에서 시연. 전체 삭제는 확인 문구 입력 후 disposable 앱 데이터에서 실행 → 확장 옵션에서 큐 삭제 → 앱 결과 완료.
- **내레이션:** “필요한 범주만 지울 수 있습니다. API 키, 웹 리더 로그인 쿠키, 재생성 가능한 캐시, 학습 데이터, 이 기기의 전체 Language Miner 데이터를 구분합니다. 전체 삭제는 DB, 파일, 화면 저장소, 키, 쿠키, 캐시, 확장 큐가 남았는지 확인한 뒤에만 완료입니다. 확장이 꺼져 있으면 앱 데이터 삭제 뒤에도 큐 확인이 대기 중일 수 있습니다. Chrome 확장 옵션에서 모두 삭제하고 앱의 확인까지 기다립니다. 직접 다른 폴더에 저장한 백업 파일은 전체 삭제가 지우지 않습니다.”
- **화면 자막:** `삭제 → 잔존 검사 → 확장 확인 → 완료` / `별도 백업 파일은 직접 관리`
- **캡처 확인:** 실제 키나 쿠키 값을 만들지 말고 canary 항목만 사용합니다. 결과에는 원시 경로가 없어야 합니다.
- **개인정보 가림:** 모든 경로, 키, 쿠키, 개인 카드·대화.
- **B-roll 매핑:** `docs/site/assets/app-images/ko/27-privacy-delete.webp`, `docs/site/assets/app-images/ko/29-extension-queue-clear.webp`, `docs/site/assets/app-images/ko/30-extension-queue-clear-confirm.webp`, `docs/site/assets/app-images/ko/31-extension-queue-cleared.webp`.

## 장면 9 — 제거, 문제 해결, 안전한 신고 (18:15–20:00)

- **클릭·입력:** 트레이 → 종료 → Windows 설정의 앱 제거 화면까지 이동하되 최종 제거는 QA VM에서만 실행. 공개 이슈와 비공개 보안 신고 경로를 비교합니다.
- **내레이션:** “창의 X는 앱을 트레이로 숨길 수 있습니다. 완전히 끝낼 때는 트레이의 종료를 사용합니다. Windows 제거가 사용자 학습 데이터를 자동으로 모두 지운다고 가정하지 마세요. PC를 넘기기 전에는 앱의 전체 삭제와 잔존 확인을 마친 뒤 제거합니다. 일반 버그는 공개 이슈로 보내되 이름, 이메일, API 키, 전체 경로, 대화 원문을 먼저 지웁니다. 키 노출, 개인정보 유출, 보안 취약점은 공개 이슈에 올리지 말고 SECURITY 문서의 비공개 절차를 씁니다.”
- **화면 자막:** `전체 삭제 확인 → 완전 종료 → 제거` / `보안·개인정보 문제는 비공개 신고`
- **캡처 확인:** SmartScreen 우회법, 키 원문, 로컬 사용자 경로가 영상에 들어가지 않는 최종 프레임 검사를 합니다.
- **개인정보 가림:** Windows 계정, 설치 경로, GitHub 계정·이메일.
- **B-roll 매핑:** `docs/install-windows.ko.md`, `SECURITY.md`.
- **공개판 추가 촬영:** 실제 설치판에서 창 닫기와 트레이 종료의 차이, 앱 내 전체 삭제 완료, Windows 제거 순서를 개인 계정·경로가 보이지 않게 촬영합니다.

# 업로드 전 최종 체크리스트

- [ ] 5편 모두 같은 `I’m running a little late.` 예문과 같은 테스트 프로필을 사용했다.
- [ ] 튜토리얼 샌드박스와 실제 웹 리더를 혼동시키는 대사가 없다.
- [ ] Web Reader를 읽기 카드 흐름으로 설명하고 듣기·말하기 카드의 전용 화면을 보여 줬다.
- [ ] Today, 루틴, 미션, 다이아 획득과 수령을 1편에서 빠짐없이 보여 줬다.
- [ ] 세 카드, 저장 전 본문 편집, 기본 샘플 영구 삭제를 정확히 설명했다.
- [ ] 프로필, 문서 최근 기록, 북마크, 용어집, 이중언어 책, 내보내기 기록을 실제 화면으로 보여 줬다.
- [ ] 듣기 루프, 영상 리더, 번역, TTS, 동기화 폴더를 보여 줬다.
- [ ] Life Mining 자동 수집이 기본 OFF이고 확장 큐 삭제 위치가 Chrome 옵션임을 말했다.
- [ ] 캐릭터챗의 외부 전송 항목과 원격 이미지 확인을 말했다.
- [ ] 공식 PlayZone 세 게임의 소스·라이선스·다운로드 크기를 최신 Release와 대조했다.
- [ ] Gemini 첫 연결 동의와 실제 작업 사전 확인을 별도 장면으로 보여 줬다.
- [ ] 비용 가드를 실제 청구 한도라고 부르지 않았다.
- [ ] ChatGPT 웹 수동 방식이 계정·쿠키에 접근하지 않고 자동 생성에 쓰이지 않는다고 말했다.
- [ ] 백업 제외 항목, 세 복원 방식, 안전 백업과 롤백을 보여 줬다.
- [ ] 전체 삭제 뒤 확장 큐까지 잔존 확인하고, 제거 프로그램만으로 삭제된다고 말하지 않았다.
- [ ] 모든 키, 이메일, 계정, 경로, 개인 대화, 결제 정보가 최종 영상과 자막에서 제거됐다.
- [ ] 존재하지 않는 Release·Discord·다운로드 주소를 약속하지 않았다.
