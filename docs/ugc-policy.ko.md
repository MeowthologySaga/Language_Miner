# Language Miner UGC 정책

[English](ugc-policy.en.md) · [제작자 가이드](creator-guide.ko.md) · [개인정보 안내](privacy.ko.md)

- 대상 버전: 공개 베타 [v0.1.0-beta.1](https://github.com/MeowthologySaga/Language_Miner/releases/tag/v0.1.0-beta.1)
- 최종 수정: 2026-07-15

## 1. 적용 범위

이 정책은 Language Miner에서 불러오는 캐릭터팩, 캐릭터 카드와 PlayZone Game Pack, 그리고 사이트의 UGC 콘텐츠 광장에 적용됩니다.

앱 코드는 GPL-3.0-only이지만 UGC의 이야기, 이미지, 음성, 게임 코드와 데이터에 앱 라이선스가 자동으로 적용되지는 않습니다. 각 제작자는 UGC manifest와 배포 페이지에 별도 라이선스를 표시해야 합니다.

## 2. 배포와 발견

- **권장 원본:** 제작자가 관리하는 GitHub Releases. 버전, 소스, 체크섬과 변경사항을 함께 남기기 좋습니다.
- **UGC 콘텐츠 광장:** 사이트는 파일을 직접 받지 않고 캐릭터 카드와 PlayZone 게임을 나눈 GitHub Issue Form으로 공개 원본 링크를 접수합니다. 제출 글과 링크는 공개되지만, 출처·라이선스·기술 정보를 확인해 `ugc-ready`가 된 항목만 일반 사용자용 카탈로그에 등록합니다.
- **공식 Discord(아직 미개설):** 향후 개설되면 콘텐츠를 발견하고 의견을 나누는 커뮤니티로 사용합니다. 현재 공식 초대 링크는 없습니다. 파일 원본은 GitHub Release 등 지속 가능한 출처로 연결하는 방식을 권장합니다.
- **Google Drive 등 외부 링크:** 미검증 외부 링크로 표시합니다. 링크 소유자와 내용이 바뀔 수 있습니다.

Issue 등록, `ugc-ready` 표시, Discord 게시물, 별표, 반응이나 다른 사용자의 추천은 Language Miner 프로젝트가 안전성, 품질, 합법성이나 저작권을 절대 보증한다는 뜻이 아닙니다. 앱은 가져오기·설치 시 기술 검사를 다시 수행합니다.

도배·중복·위험 링크 제한과 공격 시 GitHub Interaction limits 사용법은 [UGC 운영·도배 방어 가이드](ugc-moderation.ko.md)를 따릅니다. 자동 검사는 제출 파일을 다운로드하거나 실행하지 않으며, 자동 결과만으로 `ugc-ready`를 부여하지 않습니다.

## 3. 향후 공식 Discord 콘텐츠 기준

공식 Discord가 개설되면 일반 이용가 콘텐츠만 허용할 예정입니다.

허용되지 않는 예:

- 노골적인 성적 표현, 음란물 또는 성적 착취;
- 과도하거나 사실적인 유혈·고문 묘사;
- 미성년자를 성적으로 묘사하거나 위험하게 만드는 콘텐츠;
- 혐오, 괴롭힘, 신상 공개와 표적 위협;
- 불법 행위를 직접 조장하거나 실행을 돕는 콘텐츠;
- 악성 파일, 자격 증명 탈취, 추적 또는 검증 우회 시도;
- 배포 권리가 없는 책, 영화, 게임, 음악, 캐릭터나 이미지;
- 제작자·출처·라이선스를 속이는 콘텐츠.

교육적·역사적 맥락이 있어도 일반 이용가 공간에 부적절하면 제거될 수 있습니다. 반복적인 위반은 링크 삭제, 게시 제한 또는 커뮤니티 참여 제한으로 이어질 수 있습니다.

## 4. 외부에서 받은 콘텐츠

공식 커뮤니티 밖의 콘텐츠는 앱이 내용 등급을 판정하거나 성인·폭력 표현을 심사하지 않습니다. 사용자는 자신의 지역 법률, 연령, 환경과 위험 허용 범위에 따라 직접 판단해야 합니다.

내용 심사를 하지 않는다는 것이 다음을 뜻하지는 않습니다.

- 불법 콘텐츠를 허용하거나 보증함;
- 저작권 또는 상표 사용 권리를 보증함;
- 외부 링크를 공식 추천함;
- 기술 보안 검사를 생략함.

출처와 관계없이 모든 UGC에 같은 기술 검증과 실행 제한을 적용합니다.

## 5. 기술 검증 상태

| 상태 | 뜻 | 실행 |
| --- | --- | --- |
| `quarantined` | 가져왔지만 검사·승인이 완료되지 않음 | 불가 |
| `blocked` | 안전 계약 위반 또는 치명적 오류가 있음 | 불가 |
| `warning` | 실행 전 해결해야 할 경고나 불완전한 계약이 있음 | 불가 |
| `ready` | 현재 형식과 기술 검사를 만족함 | 가능 |
| `trusted_official` | `ready` 조건을 만족하고 앱의 공식 카탈로그 ID·해시와 일치함 | 가능 |

`ready`는 악성 의도가 절대 없거나 콘텐츠가 좋은 품질이라는 인증이 아닙니다. 알려진 형식·파일·권한 규칙을 만족했다는 기술 상태입니다.

검사 대상에는 다음이 포함됩니다.

- manifest 형식, 버전, 제작자, 출처 URL, SPDX 라이선스;
- 파일 해시와 선언되지 않은 파일;
- ZIP 경로 탈출, 심볼릭 링크, 크기·개수·압축률, CRC와 ZIP 구조;
- HTML 진입점과 CSP;
- 요청 권한, 네트워크 기본 거부, Host API 사용;
- 다이아 동작의 id, 금액, 사유와 중복 차감 방지;
- 캐릭터팩의 실행 가능 필드와 안전하지 않은 이미지 참조.

## 6. 캐릭터팩

캐릭터팩은 데이터 전용입니다.

허용되는 기본 범위:

- JSON 캐릭터 설정;
- 안전한 로컬 래스터 이미지;
- 평문 또는 안전하게 렌더링되는 Markdown 설명.

HTML, JavaScript, 실행 파일, 원격 스크립트와 자동 원격 이미지 로드는 허용하지 않습니다. 캐릭터 프롬프트가 사용자 데이터, 시스템 지시, API 키나 다른 비밀을 빼내도록 설계되어서는 안 됩니다.

클라우드 캐릭터챗을 실행하면 캐릭터 설정, 최근 대화, 새 메시지와 카드 힌트가 공급자에게 전송될 수 있습니다. 제작자는 설명에서 필요한 맥락 범위를 과장 없이 밝혀야 합니다.

## 7. Game Pack

Game Pack은 코드가 포함될 수 있으므로 격리된 HTML 실행 환경에서만 엽니다.

- 진입점은 팩 내부의 `.html` 파일이어야 합니다.
- Electron·Node API와 앱 DB에 직접 접근할 수 없습니다.
- 네트워크는 기본 거부되며 첫 베타에서는 UGC 네트워크 권한을 지원하지 않습니다.
- 저장과 로컬 보상 사용은 선언된 Host API만 이용합니다.
- 다이아 차감은 manifest의 id·금액·사유와 정확히 일치하고 사용자 확인 및 idempotency key를 사용해야 합니다.
- 팩별·전체 저장 용량 제한을 따릅니다.

## 8. 제작자 책임

제작자는 다음을 보증할 수 있어야 합니다.

- 자신이 모든 파일을 배포할 권리가 있음;
- 제작자, 원본 URL, 생성 방식과 라이선스가 정확함;
- 제3자 고지와 표시 의무를 지킴;
- 비밀 키, 토큰, 개인 대화, 로컬 경로와 개인정보가 없음;
- 업데이트에서 `id`, `lineageId`와 SemVer를 올바르게 유지함;
- 요청 권한과 다이아 동작을 최소 범위로 선언함;
- 알려진 취약점과 호환성 문제를 숨기지 않음.

AI로 만든 자산도 자동으로 자유롭게 재배포할 수 있는 것은 아닙니다. 사용한 도구의 약관, 학습·참조 소스, 실존 인물과 상표·캐릭터 권리를 확인하고 생성 방식을 기록하세요.

## 9. 사용자 책임

- 제작자와 원본 URL을 확인합니다.
- 설치 전 해시, 라이선스, 권한과 경고를 읽습니다.
- 낯선 Google Drive 링크보다 버전이 고정된 GitHub Release를 우선합니다.
- 민감한 학습 데이터가 있는 프로필에서는 출처가 불명확한 팩을 사용하지 않습니다.
- 문제가 생기면 팩을 중지하고 보안 리포트와 버전을 기록합니다.
- 의심스러운 팩을 다른 사용자에게 다시 배포하지 않습니다.

## 10. 신고와 조치

- 일반 호환성 문제는 UGC 이슈 템플릿을 사용합니다.
- 저작권·상표·커뮤니티 내용 문제는 공식 커뮤니티의 비공개 운영자 채널이 공개된 뒤 그 경로로 신고합니다.
- 경로 탈출, 네트워크 우회, 정보 유출, 다이아 중복 차감 같은 취약점은 [비공개 보안 신고](../SECURITY.md)를 사용합니다.

공개 이슈에 악성 파일, 개인 데이터, API 키나 검증 우회 세부 정보를 올리지 마세요.

프로젝트는 링크나 게시물을 제거하고, 공식 추천 표시를 취소하고, 기술적으로 위험한 해시·팩 버전을 차단할 수 있습니다. 외부 저장소의 원본을 삭제할 수는 없습니다.
